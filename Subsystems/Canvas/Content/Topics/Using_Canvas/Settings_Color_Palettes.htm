Configuring Color Palettes

Canvas supports custom color palettes, which can be configured in the Canvas settings and then applied to any screen containing CygNet-aware controls.

CygNet supports a dual mode for point state color; that is, each point type allows both a background color and a foreground color to be set. The point state definition colors are configured in the CVS metadata file for each point scheme.

In CygNet-aware controls the color configuration for point state supports either background color or foreground color:

  • Typically background color is used for the closed space on a control, for example, the background area, borders and lines, grid cells, bars and rings, markers, and fills. The default background color for the standard point scheme is configured from one of the following colors: green, cyan, magenta, yellow, or red.
  • Typically foreground color is the element on the face of control, usually the text. For most point states, the default foreground color for the standard point scheme is black.

In Canvas you can create a custom color palette to override the default point state colors for any point type. If a custom color palette is selected for a screen, any CygNet-aware controls using Point State for color will apply the selected color palette in run mode.

These settings are saved in XML format as part of the global settings file (.gsf) for your Canvas application and applied to all instances of Canvas.

Configure Custom Color Palettes

Color palettes are defined in the Settings in the Backstage view and applied to individual screens. The workflow is described below.

To manage and configure custom color palettes in the Backstage view

The Backstage view is the location for globally managing and configuring color palettes.

  1. In the Backstage view, go to Settings, and click Color palettes to open the Color Palettes dialog box.

Color Palettes — Backstage view
  1. Click Add to add a new color palette. The color palettes are added sequentially and you can change a color palette's name to suit your needs.
  2. Click the name "NEW" and type a new name for your palette and press enter.
  3. Click Point scheme and select the appropriate point scheme for this palette from the drop-down menu. The point scheme defines the point types, point alarms, point statuses, and default colors for a CygNet domain.
  4. Click (Edit color palette) to open the Color Palette for <Name of Palette> dialog box for the selected color palette.
  5. In the Color Palette for <Name of Palette> dialog box configure new colors for background and foreground for each point state for each point type.

Color Palette for <Name of Palette>— Backstage view
  1. Select the point type and point state you want to override for Background and Foreground.
  2. A read-only override check box is displayed next to the color box. When you change a color this check box becomes enabled.
  3. Click to select a new color using the color picker. When you change a color the point state name becomes bolded.
  4. Click (Revert colors) to undo any changes and return to the default color.
  5. Click OK to save any new color settings.
  6. Or click Cancel to return to the Color Palettes dialog box (colors will not be changed).

The following table describes the columns on the Color Palette for <Name of Palette> dialog box.

Column Description

Type

The Type property lists the point types used in the selected point scheme. The point types configured for the selected point scheme for your CygNet system. The default CygNet Standard Point Scheme has eight point types: Analog Input & Analog Output, Digital Input & Digital Output, Enumeration Input & Enumeration Output, and String Input & String Output. Select the type you want to edit.

Point State

The Point State property lists the point states used in the selected point type. Point state is the highest precedented state for a point record as defined for each point scheme. The point state for a point record is based on 48 System, Configuration, and User status bits, which are associated with the four point types (Analog, Digital, String, Enumeration). A point record may contain up to 48 status bits, which are used to provide point status information about the point record in a CVS and the ELSALM. A point can be in multiple states at the same time, for example, in High Warning and High Alarm; however, the state defined to be the most severe is the one that is used for the point state (i.e., High Alarm, in this case). Select the point state you want to edit.

Background

This column shows the default background color specified for the selected point state for the selected point type for the selected point scheme.

  • A read-only override check box is displayed next to the color box. When you change a background color this check box becomes enabled.
  • Click to select a new color using the color picker.
  • When you change a background color the point state name becomes bolded.

Foreground

This column shows the default foreground color specified for the selected point state for the selected point type for the selected point scheme.

  • A read-only override check box is displayed next to the color box. When you change a foreground color this check box becomes enabled.
  • Click to select a new color using the color picker.
  • When you change a foreground color the point state name becomes bolded.

Revert

  • Click (Revert colors) to undo any changes and return to the default color.

OK / Cancel

Once your colors are configured, click OK to return to the Color Palettes dialog box. Or click Cancel to abort and close the Color Palette for <Name of Palette> dialog box.
  1. Click (Delete color palette) to remove a selected color palette.
  2. Once your color palette is configured, click OK to save the configuration to the global settings file.
  3. Or click Cancel to abort and close the Color Palettes dialog box.
To configure a custom color palette in the Properties pane

It is easy to apply a custom color palette to all CygNet-aware controls on a screen.

  1. Configure a custom color palette as described above.
  2. Select the custom color palette in the Color palette property in the Properties pane for your screen.
  3. Configure other screen properties as desired.
  4. Add the CygNet-aware controls to the screen.
  5. Configure the color configuration properties on each control to use Source > Point State. See the individual color properties for the control you are adding to the screen for more configuration information.
  6. In run mode the CygNet-aware controls will use the point state colors configured in the custom color palette selected in step 2.
