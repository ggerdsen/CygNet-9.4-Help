Configuring Alarm Box Attributes

These attributes specify the text that will appear in the three items in the alarm box on the notifier. The text can be a static label or dynamic text based on point or facility attributes. Dynamic text is built using tokens, which can represent real-time CVS point properties, point configuration properties, and facility properties. Each alarm box will expand in width to display any specified text.

Alarm box examples

These settings are saved in XML format as part of the global settings file (.gsf) for your Canvas application and applied to all instances of Canvas.

Configure Alarm Attributes

  • In the Backstage view, go to Alarm Notifier, Alarm attributes, and for each item in the alarm box, click to configure the text strings using the CygNet Property String dialog box.

Alarm attributes configuration

The default tokens for the three items are facility description (%FacilityDescription%), point value (%Value%), and point description (%Description%); but you can define any desired static text or tokens.

Alarm Box Preview

The contents of the alarm box will be previewed in the box below the configuration items.

Configure Facility Link

The top left item in the alarm box will link to a screen of your choice for the associated facility. The navigation mode is determined by the Navigation by Facility feature.

Perform the following steps to configure the facility attribute name, the path to the associated screen in the facility attribute, and the facility navigation mode:

  1. Define the name of the facility attribute (and any optional facility record properties) on the Settings page of the Backstage view. See Canvas Facility Screen Navigation for instructions on how to do this.
  2. Configure the path of the associated screen in a facility attribute for the facility. See Canvas Facility Attribute for instructions on how to do this.
  3. Configure the facility navigation mode for the screen displaying the Alarm Notifier. See Screen Navigation Mode for instructions on how to do this.
  4. Create the associated screen referenced in step 2.
