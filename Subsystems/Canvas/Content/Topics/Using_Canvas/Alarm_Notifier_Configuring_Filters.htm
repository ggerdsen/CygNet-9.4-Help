Configuring Alarm Filters

Use the Alarm Filter dialog box to create custom alarm filtering rules to limit the alarms displayed in the alarm notifier.

These settings are saved in XML format as part of the global settings file (.gsf) for your Canvas application and applied to all instances of Canvas.

Configure alarm filtering rules

The filter alarms feature is useful when a site has numerous alarms and you want to single out a few for viewing or those that meet a certain criterion. Alarms can be filtered based on a variety of attributes and values. You can filter alarms using alarm states, real-time service values, point states, point attributes, and facility attributes. These attributes allow you to set criteria so that only those alarms meeting all, not part of, the criteria will be displayed. Rules may be (AND)’ed or (OR)’ed to create a single rule filter, or options are available to combine multiple single rules to (AND) and (OR) rules together to create compound rules.

In the Backstage view, go to Alarm Notifier, and click Alarm filter to open the Alarm Filter dialog box.

Alarm filter configuration
  1. Click Add to add a new filter rule, which invokes the Rules Definition dialog box where you can configure the Data Item, Operator, Comparison Type, Comparison Item, Rule Name, and Enable Rule options.
  2. Click Delete to delete a selected filter rule.
  3. Move the filter rules up and down the filter hierarchy tree with the << and >> buttons.
  4. Export and import filter rules as necessary using the Export Rules and Import Rules buttons.
  5. Click OK to accept the filter rule and close the dialog box.
  6. Click Cancel to close the dialog box without configuring or changing a filter rule.
