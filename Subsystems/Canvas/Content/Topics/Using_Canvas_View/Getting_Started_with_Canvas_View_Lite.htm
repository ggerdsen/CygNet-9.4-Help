Getting Started with Canvas View Lite

Canvas View Lite is a read-only run-time companion application to Canvas. While Canvas View can create complex layouts of multiple screens in pinned, docked, tabbed, or floating states, Canvas View Lite utilizes a single main screen that drives all other screen navigation. It is always fullscreen and removes all user interaction that is not part of the screen’s HMI (no file menu, toolbar, backstage view, status bar, etc.). The main screen is opened via command-line.

Both visualization applications are valuable for different HMI configurations. Because Canvas View Lite doesn’t use the docking framework that Canvas View does, it loads faster, at the cost of a far simpler UI and simpler layout options.

Canvas View Lite can only open Canvas (.can) files:

  • any Canvas screen (.can), opened directly or via hyperlink
  • any Canvas screen (.can) containing a Nested View control, which can reference a CygNet Studio screen (.csf). It does support nested CygNet Studio (.csf).

Start Canvas View Lite

Canvas View Lite must be started via the command-line interface.

Exit Canvas View Lite

Canvas View Lite can only be exited via the Task Manager.
