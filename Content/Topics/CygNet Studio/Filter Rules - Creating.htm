Creating Filter Rules

To create new rules in CygNet Studio, enter Edit mode, and place the desired ActiveX control on your screen. Open the Properties dialog box for the control, and then browse to the button or Filter page to bring up the Filter dialog box.

In Run mode, right-click on the control and select Filter from the context menu.This will invoke the Filter dialog box with the Tree View and Text View pages displayed.

The Tree View page displays the current filter rules, and lets you add, edit, delete, and reorder an existing rule.

  • Double-click on the root of the rule tree to change the rule type and its properties.

The Text View page is used as a read-only display of the rules in a multi-line textual representation of the filter rules. You cannot add, edit or modify rules using the Text View.

Both pages allow the export and import of rules in .xml format.

Parameter Description

Add

Click to add a new filter rule. This invokes the Rules Definition dialog box. See Adding Filter Rule Definitions for information about adding a new filter rule.

Delete

Click to delete a selected filter rule.

<<

Click to move a selected filter rule up the filter hierarchy tree.

>>

Click to move a selected filter rule down the filter hierarchy tree.

Export Rules

Click to export filter rules to an XML file.

Import Rules

Click to import filter rules from an XML file.
