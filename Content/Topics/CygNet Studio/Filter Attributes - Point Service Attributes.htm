Filter Attributes - Point Attributes

The following filter attributes represents properties from the point configuration record. They are available for use throughout CygNet Software.

  • Point: General
  • Point: History
  • Point: Point Reference
  • Point: Application
  • Point: Alarm (PNT)
  • Point: Alarm (Scheme 0)
  • Point: Scaling
  • Point: HyperPoint

Point: General

See PNT Editor - General Page for more information.

Attribute Attribute Name Description User Values

Active Status

activestatus

This attribute is used for filtering on the status of the point.

0 (no) or 1 (yes)

Allowed Updates

allowedupdates

This attribute is used for filtering on the setting of update values for the alarm.

B (Update All)

T (Internal /External)

C (Internal User)

I (Internal)

X (External User)

E (External)

U (User)

N (Updates None)

Allowed Updates Description

allowedupdates_desc

This attribute is used for filtering on a description of the Allowed Updates; for example, "Update All"

Text

Calc Parameter Error Exists

calcparmerror

This attribute is used for filtering on the alarm that has a calculation parameter error. A calc parm error is set by the PNT if the point contains invalid calculation parameters.

0 or 1

Comment

comment

This attribute is used for filtering a comment included in the alarm record. Check spelling; when possible, use the wildcard *.

Text

Display Attribute

displayattribute

This attribute is used for filtering on the display attribute for the alarm condition.

Text

Display Attribute Desc

displayattribute_desc

This attribute is used for filtering on the display attribute description for the alarm

Text

Facility ID

facilityid

This attribute is used for filtering on the Facility ID of the alarm.

Alphanumeric

Full Tag

tagfull

This attribute is used for filtering on the full tag of the alarm; for example, SITE.SERVICE.POINTID:LONGID.

Alphanumeric

General Point Config Error Exists

generalconfigerror

Attribute is used if a general point configuration error exists in the alarm. A general config error is set by the PNT for a variety of reasons, for example:

  • if the primary and secondary units are not convertible
  • if there are invalid entries configured for History reporting settings

0 or 1

GNS ID Entry Error Exists

gnsidentryerror

This attribute is used for filtering on a GNS ID entry that is in error for the alarm. This option can be useful for troubleshooting. For instance, you can filter to show those alarms that have a problem with a GNS ID.

A GNS ID entry error is set by the PNT if a GNS ID is configured on a point which does not exist in the GNS.

0 or 1

Initial Type

initialtype

This attribute is used for filtering on the initial type for the alarm when it is accessed (for example, (R)estored from the persist file or (I)nitialized.)

Text

Initial Type Desc

initialtype_desc

This attribute is used for filtering on a text description of the initial type.

None, Restore, Service Configuration, Specified Value

Initial Value

initialvalue

This attribute is used for filtering on the initial value for the alarm.

Alphanumeric

Key Field Entry Error Exists

keyfieldentryerror

This attribute is used for filtering on a key field entry error that exists in the point’s alarm condition. A key field entry error is set by the PNT if a referenced point does not exist. Does not apply to the OPCIS.

0 or 1

Long Description

longdescription

This attribute is used for filtering on the long description of the alarm.

Text

Long Point ID

pointidlong

This attribute is used for filtering on the long point ID of the alarm, i.e., Facility_UDC, for example, METER04_LTANK.

Alphanumeric

Long Tag

taglong

This attribute is used for filtering on the long tag of the point’s alarm condition, which would be SITE.SERVICE:FacilityID_UDC.

Alphanumeric

Manual Input Enabled

manualinputenabled

This attribute is used for filtering on whether or not manual input is enabled for the alarm.

0 (no) or 1 (yes)

Point Data Type

pointdatatype

This attribute is used for filtering on the point data type for the alarm (for example, AI for Analog Input, AO for Analog Output, DO for Digital Output, etc.)

AI, AO, DI, DO, EI, EO, SI, SO

Point Data Type Desc

pointdatatype_desc

This attribute is used for filtering on the description of the point data type for the alarm (for example, Analog Input, Digital Output, etc.)

Text

Point Has Comment

hascomment

This attribute is used for filtering on the alarm that has a comment.

0 or 1

Point ID

pointid

This attribute is used for filtering on the alarm’s short point ID (for example, 8 characters, such as 00000255).

Numeric

Point Scheme

pointscheme

This attribute is used for filtering on the point scheme used for describing point states and alarm conditions. The CygNet Standard Point Scheme is 0.

0 (or user-customized)

Point Scheme Description

pointschemedesc

This attribute is used for filtering on the name of the global point scheme for an alarm. The default is CygNet Standard.

CygNet Standard (or user-customized)

Security Application

securityapp

This attribute is used for filtering on the security application for the alarm.

Text

Service

service

This attribute is used for filtering on the service for the alarm, for example, UIS, SVCMON, OPCIS.

Text

Site

site

This attribute is used for filtering on the site for the alarm.

Text

SiteService

siteservice

This attribute is used for filtering on the Site.Service for the alarm.

Text

Spare

spare

This attribute is used for filtering on an attribute internal to the CygNet application.

Text

System Desc

system_desc

This attribute is used for filtering on the System description for an alarm.

Text

System Name

systemname

This attribute is used for filtering on the System name for an alarm.

Text

Tag

tag

This attribute is used for filtering on the tag for the point; for example, Site.Service.PointID.

Text (alphanumeric)

Uniform Data Code

uniformdatacode

This attribute is used for filtering on the uniform data code (UDC) for an alarm.

Text (alphanumeric)

Uniform Data Code Desc

uniformdatacode_desc

This attribute is used for filtering on the point’s UDC description for an alarm.

Text
