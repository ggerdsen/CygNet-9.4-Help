CygNet Filter Rule Types

CygNet provides a filter rule tree view tool so that you can create custom filtered rules to assist in displaying data to the end user. Any CygNet data point attributes currently available can be used as a comparison type for the filter rule.

Single Rules

Single rules are treated as an independent single rule, adding one rule at a time for each Parent. The following is an example of a text view of a single rule:

( Alarm Category = Alarm Condition, Case Insensitive )

Compound Rules

Compound rules are single independent rules created in the same fashion but are grouped under the same parent and can be configured with any combination of (AND or OR) statements together. The following is an example of a text view of a compound rule:

( Alarm Category = Alarm Condition, Case Insensitive )

OR

( Alarm Category = Alarm Hidden, Case Insensitive )

A rule or rule(s) for a level has two purposes: the first is to determine which records to include, and the second is to link the records to the parent level.

Combining Rules

Rules may be (AND)’ed or (OR)’ed to create a single rule filter, or options are available to combine multiple single rules to (AND) and (OR) rules together to create compound rules.

Parameter Description

AND

All Rules must be met.

OR

Only a single rule must be met.

Single Rule

Single (AND) or (OR) Rule to filter data.

Compound Rule

Multiple Single (AND) or (OR) rules combined to filter data.
