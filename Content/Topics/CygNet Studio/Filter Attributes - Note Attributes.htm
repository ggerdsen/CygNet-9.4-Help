Filter Attributes – Note Attributes

The following filter attributes can be used to filter notes in a Trend Tool. The * wildcard character may be used in various user values.

See Trend Tool – Configuring Display Information and Notes.

Attribute Description User Value

Note Association's Category

This attribute is used to filter on the note's association being either a facility or point.

FACILITY or POINT

Note Association

's ID

This attribute is used to filter on an associated item's unique identifier.

User-defined ID for a facility and system-defined ID for a point.

Note Association's Service

This attribute is used to filter on an associated item's service name.

Valid service name

Note Association's Site

This attribute is used to filter on an associated item's site name.

Valid site name

Note Attribute 1-5

These attributes are used to filter on a value that corresponds with the Attribute 1-5 field of a note.

Text

Note Table Attribute 1-5

These attributes are used to filter on a value that corresponds with the Table Attribute 1- 5 field of a note.

Valid table entry name

Note Yes/No Attribute 1-5

These attributes are used to filter on whether a value that corresponds with the Yes/No Attribute 1 field of a note.

Yes or No

Note Author

This attribute is used to filter on the name of the author of a note as it is recorded in the Note Service.

Text

Note End Time

This attribute is used to filter on the end time for a ranged note.

Valid date and/or time. See Date and Time.

Note Start Time

This attribute is used to filter on the start time for a note.

Valid date and/or time. See Date and Time.

Note Summary

This attribute is used to filter on text in the Summary field of a note.

Text

Note Type

This attribute is used to filter on the user-defined text in the Type field of a note.

Text
