Logging

Logging in CygNet is handled by instances of event logging services (ELS or ELSALM) that are designated per CygNet service to retain log file information. Additionally, events can be logged to the CygNet service log files instead of the designated event logging service.

CygNet system issues, either with the service itself or with its operational functions, are logged to the selected log file location(s), which can then be accessed for analysis and troubleshooting purposes. Log files handled by and ELS are accessible via CygNet Explorer whereas log files handled by the CygNet service are accessible via the service file on your host server.

Logging for each service is configured in its service configuration (.cfg) file, found in the CygNet\Services\[service name] folder on your host server. See Configuring Logging for more information.

Set ELS retention periods in the service configuration (.cfg) file of the selected logging service.

To control ELS database size, consider what types of events to log to available logging locations.
