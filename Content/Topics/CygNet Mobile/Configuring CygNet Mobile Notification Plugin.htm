Configuring the CygNet Mobile Notification Plugin

Once the CygNet Mobile Notification Plugin is installed, configure the plugin to interact with your CygNet software installation and the CygNet Mobile Application Suite. After configuring the CygNet Mobile Notification Plugin, you can receive and acknowledge CygNet notifications over your mobile device using the CygNet Operator application.

Configuring CygNet Mobile Notification Plugin can occur using one of the following methods.

  • Add an address to an existing Event record
  • Add an address to an existing Group record
  • Create a new Address record

See Best Practice for Setting up Notification Record Types for more information about planning your GNS records and record types.

Adding an Address to an Existing Event Record

To add a primary address record to an existing Group record, use the following procedure.

To Add a Primary Address to an Existing Event Record for the CygNet Mobile Notification Plugin

  1. In CygNet Explorer, open the General Notification Service (GNS) and double-click on the Event record to use. The Properties for Entry dialog box will appear.
  1. On the Addresses page, click New to access the Notification Address dialog box.
  1. In the Type field, use the drop-down menu to select Reference address.
  2. In the Address field, click ... to access the Select GNS ID dialog box. Select the primary address record to use (MOBILE_PRIMARY), and click OK.
  3. Optionally in the Description field, type descriptive text to help you identify the record in the future.
  4. Optionally click to select Propagate resend rules down, and optionally Report Alarm Clear. See Configuring an Address Record for more information.
  5. Optionally click to select Ignore all previous resend rules. See Configuring an Address Record for more information.
  6. Optionally select to require acknowledgment. Select the Acknowledgment Required check box and continue to the next step. See Configuring an Address Record for more information.
  7. Optionally select to resend notifications that are not cleared. See Configuring an Address Record for more information.
  8. Optionally schedule blackouts if required. See Configuring Blackouts for more information.
  9. Click OK to close the Notification Address dialog box.
  1. In the Properties for Entry dialog box, click OK to save the new notification address for the Event record.

Creating a New Address Record

To create a new address record, use the following procedure. You will be using the Notification Plugin Manager found in the General Notification Service (GNS) to aid in configuration.

To Create a New Address Record for the CygNet Mobile Notification Plugin

  1. In CygNet Explorer, open the General Notification Service (GNS) and right-click in the white space of the pane to access the context menu. Select View Notification Plugin Manager to access the Notification Plugin Manager dialog box.
    • If the CygNet Mobile Notification Plugin is listed in the Manager, select it and then click Edit to access the Edit CygNet Mobile Notification Plugin dialog box to manage the mobile plugin settings.
    • If the CygNet Mobile Notification Plugin is not listed in the Manager, click Add to access the Add New Notification Plugin dialog box to create the mobile notification plugin settings.
  1. In the Edit... or Add... dialog box, specify the following values to configure the CygNet Mobile Notification Plugin settings.
  1. In the Address type field, type ZM.
  2. In the Notification name field, type CygNet Mobile Notification Plugin.
  3. In the Full notification file path field, the address is read-only to display the directory where the plugin assembly files are stored. It will be populated in part with values provided for the Relative notification file path value you enter in the next field, e.g. C:\CygNet\Services\GNS\Plugins\CygNetMobile\CygNetMobileGNSPlugin.dll.
  4. In the Relative notification file path field, define where the custom plugin assembly files are stored relative to the GNS (requires a dedicated directory for each plugin). Type in the location e.g. Plugins\CygNetMobile\CygNetMobileGNSPlugin.dll.
  5. In the Regular expression field, specify acceptable addresses. Enter your primary, and optionally secondary and tertiary, record addresses. Separate multiple values with a pipe, e.g. PrimaryAddress|SecondaryAddress|TertiaryAddress. Default value is .* or all valid addresses.
  6. In the Max parallel notifications field, type the number of push notifications to allow at the same time. Maximum value is 8; default value is 1.
  7. In the Notification time limit field, type the number of seconds to allow for the notification delivery. Value range is 60 seconds (1 minute) - 3600 seconds (1 hour). Default value is 600 seconds (10 minutes).
  8. Click Save to commit the settings and close the dialog.
  1. Validate the settings as follows.
  1. In the Notification Plugin Manager dialog box, click Validate plugins.
  2. If errors occur, resolve the issues. See Monitor Plugin Errors and Validation State, in the CygNet Notification Plugin Interface topic, for more information.
  3. When the Plugins validated successfully message appears, click Close to exit the Notification Plugin Manager.
  1. Edit the CygNet Mobile Notification Plugin configuration file as follows.
  1. Open the CygNetMobileGNSPlugin.dll.config file using the text editor of your choice, such as Notepad++. The file is located in the \CygNet\Services\GNS\Plugins\CygNetMobile directory on the CygNet server.
  2. Change the URL value by locating the following line and modifying it to point to your mobile server location:

    <add key="BaseMobileRestUrl" value="http://localhost/" />

    Note: Use forward slashes (/) for the URL values, not backslashes.

  3. Change the GUID value (MobileNotificationGuid) as follows:
  1. Open the Web.config file. The file is located in the \\localhost\Weatherford\CygNetBridge\Website\CygNetBridge directory on the Web server.
  2. Copy the mobile notification GUID values (MobileNotificationGuid) from the Web.config file into the corresponding location in the CygNetMobileGNSPlugin.dll.config file.
  1. Save your changes and close the file.

Notifications from these points will be sent to the mobile service, which will notify appropriate users based on their hierarchy assignments. See Editing Points and Point Configuration Manager Utility for more information about configuring point properties.
  1. In the PNT pane, double-click on the point you want to edit, or right-click on the point desired to access the context menu and then click Properties, to access the Properties for Point dialog box.
  2. Click Edit to access the editable fields to edit properties for the point.
  3. On the [point type] page (e.g. Analog, Digital), click the corresponding Report to GNS check box to configure each desired alarm. To specify the GNS ID value, type in MOBILE_PRIMARY_E, or click ... to select MOBILE_PRIMARY_E from the GNS IDs available in your service. This is the GNS ID you previously configured to have the MOBILE_PRIMARY address value.
  4. Click Save to save your changes and return to the service pane.

    Note: In the PNT Editor, when not in Edit mode, you can use the Previous and Next buttons to access additional points to perform more edits without returning to the service pane.

  5. Click Refresh to retrieve the new point information.
