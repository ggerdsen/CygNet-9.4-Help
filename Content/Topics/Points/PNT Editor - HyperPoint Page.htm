HyperPoint Properties

This page contains the script used by the point. The HyperPoint page is shown when the HyperPoint check box (included on the General page) is set to 1 (Yes).

The Activate Script flag must be set to 1 (Yes) for scripting on the point to be active. In addition, the current value service of the point must be a service for which scripting is enabled.

Note: You cannot import script into a point. The field is read-only. However, you can use the CygNet Point Script Copy utility (PointCfgScriptCopy.exe) to load script into a point. See CygNet Point Script Copy Utility for more information.

Property Description Default Value

HyperPoint type

Specifies the type of HyperPoint. Possible values are as follows.

  • Standard HyperPoint (S) — A Standard HyperPoint is a self-contained HSS point, that does not call or include any other script.
  • Library HyperPoint (L) — A Library HyperPoint is an HSS point that contains script designed to be used by other HyperPoints, called Library User HyperPoints. It can be called from multiple other HyperPoints, which access a temporary copy of its script. This means that any changes made to a Library HyperPoint script will apply to all its users. Library HyperPoints are not recommended for script processing.
  • Library User HyperPoint (U) — A HyperPoint that uses a Library HyperPoint is called a Library User HyperPoint. Multiple instances of Library User HyperPoints can use the same Library HyperPoint. Library User HyperPoints are recommended for script processing.
