Editing Alarm Delays

Points can be configured to report an alarm to the CAS or the GNS. When this is the case, you can also optionally configure an alarm reporting delay for the point. If configured, the reporting of the alarm is delayed for the specified period of time. To use Point Configuration Manager to set or edit an alarm delay, consider the delay value, its units, and how the settings are applied.

Applying Alarm Delay Settings

To specify alarm delay settings in the Point Configuration Manager, you must define both an integer value for the reporting delay and a time increment value for the delay units to fully represent the type of delay.

Example

Specify a Reporting Delay of 1, and Report Delay Value Units of S, to result in a delay of 1 second. If you want to change the delay to be 1 hour in this context, you can either reset the Reporting Delay value to 3,600 (seconds), or keep the Reporting Delay value of 1 and change the Reporting Delay Value Units value to H.

Note: Coordinate both settings when editing alarm delay values. When you change the Delay Units, the value is not scaled to the new units. If you had a delay value of 3,600 and changed the Delay Units to H instead of S, the delay would then be 3,600 hours.

Editing Reporting Delays

The delay in alarm reporting is enabled by specifying a value for a Reporting Delay configurable bit. This property is found in the Point:Alarm attribute category, and serves as the time value for the delay. A value of 0 means the delay is not set.

Possible values are integers.

Example

Specify a value of 8 for the configurable bit Reporting Delay. The delay is then enabled, and the time value is 8. To disable the delay, reset the configurable bit Reporting Delay value to equal to 0.

To Edit a Point Alarm - Reporting Delay

  1. In the Point Configuration Manager utility, use the filter to include the desired point in the results list.
  2. Select the point to modify and then click Edit to access the Edit Point Attribute dialog box.
  3. Click Select Attribute to access the Select Attribute to Edit dialog box.

  1. In the Select Attribute by Category box, select the Point: Alarm category from the drop-down menu to populate the attribute list.
  2. Select the desired Configurable Bit Reporting Delay from the list, and click OK. It will appear in the Current Value(s) list.
  1. Enter a New Value for the selected reporting delay. Valid values are integers.
  2. Click Update Point Records to commit the change.

Editing Delay Units

The other necessary value when setting a delay in alarm reporting is the Report Delay Value Units. For each point, the delay can be specified in hh:mm:ss (hours, minutes, seconds), DD:hh:mm (days, hours, minutes), or Y:DDD:hh (years, days, hours) formats. In the Point Configuration Manager utility, this property is found in the Point: Alarm attribute category, and serves as the time units value for the delay.

Possible values are as follows.

  • S = hh:mm:ss (seconds)
  • M = DD:hh:mm (minutes)
  • H = Y:DDD:hh (hours)

To Edit a Point Alarm - Report Delay Value Units

  1. In the Point Configuration Manager utility, use the filter to include the desired point in the results list.
  2. Select the point to modify and then click Edit to access the Edit Point Attribute dialog box.
  3. Click Select Attribute to access the Select Attribute to Edit dialog box.

  1. In the Select Attribute by Category box, select the Point: Alarm category from the drop-down menu to populate the attribute list.
  2. Select Report Delay Value Units from the list, and click OK. It will appear in the Current Value(s) list.
  1. Enter a New Value for the selected report delay value units. Valid values are S (seconds), M (minutes), or H (hours).
  2. Click Update Point Records to commit the change.
