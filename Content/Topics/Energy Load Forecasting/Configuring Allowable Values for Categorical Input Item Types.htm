Configuring Allowable Values for Categorical Input Item Types

Each Categorical data input item type must specify a list of allowable values.

Categorical data item values are stored in String Input points in the PNT, and the string identifier for the possible values will be the value that is stored in the point’s history. This makes the identifier the unique key to the historized values, but you can specify either the value identifier or the description for an associated input data record in the import file.

Categorical data input item types are fed into the neural network for forecasting and training with a separate input value for each allowable value for the categorical input. Because of this, once a neural network is trained with the categorical data values, it is important that the number and order of the possible values is the same when forecast data is provided. If the number or contents of the list of possible values changes, this invalidates the trained neural network and it would have to be retrained with the new definitions of the inputs. To detect this, the neural network definition will store the table of allowable values for each categorical input data item type used in training. If another training process or a forecast process occurs and the currently configured possible value table is different, the neural network will be put into an invalid state and will need to be retrained.

To Configure Allowable Values for Categorical Input Item Types

  1. Click the Input Items page of the ELF Editor.
  2. Click Add Item or Edit Item to view the Add/Edit Input Data Item dialog box.
  3. For new items select Categorical for the Input Item Type.
  4. Click Add… or Edit… to view the Add/Edit Allowable Item dialog box.
  5. Configure the value and description using the properties described below.

The properties on the Add/Edit Allowable Value dialog box are described below.

Parameter Description

Input Item Category

The Input Item Type for the new or selected Allowable Value.

Input Item Name

The Input Item Name for the new or selected Allowable Value.

Allowable Value

Specify a string identifier up to 16 characters long.

Value Description

Specify an associated value description of unbounded length.
