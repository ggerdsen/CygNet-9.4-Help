Configuring Meter/Meter Group Assignment Settings

Use the Meter/Group Settings - Assignments page to configure the following settings for the selected meter or meter group:

  • Value Assignment Type and associated rules (rollup forecast, assignment from parent, aggregation from children, expression redistribution, and balance redistribution)
  • Value Adjustment
  • Hierarchy Membership and Assignment Usage
  • Hierarchy Parent
  • Hierarchy Children and Assignments to Children (if appropriate)

To Configure Meter Assignment Settings

  1. Click the Meters/Owners page of the ELF Editor.
  2. Add a New Child or New Sibling, or double-click to Edit a configured meter.
  3. Configure the properties in the General page.
  4. Click the Assignments page.
  5. Configure the settings using the properties described below.

The properties on the Meter/Group Settings dialog box, Assignments page are described below.

Meter/Meter Group Settings

Parameter Description
Full Target ID Specifies the ID of the current target meter or meter group.

Value Assignment Type

Any node (meter or meter group) may only get its values from one source. This setting specifies the Value Assignment Type for the current target meter or meter group. Value Assignment defines the different ways in which a meter or meter group receives its forecasted values. Assignment options include the following and are described below:

  • None
  • Direct Forecast
  • Rollup Forecast
  • Assignment from Parent
  • Aggregation from Children
  • Expression Redistribution
  • Balancing Redistribution

Select the appropriate Value Assignment Type for the target and click the adjacent button to configure the rule.

The facility attribute used to store the Value Assignment Type value is configured on the System Settings page. See Configuring Facility Attribute Settings for more information.

Value Assignment Type Description

None

No gross forecast is generated.

Direct Forecast

The process by which a meter or meter group’s gross forecast value is generated by a neural network. Once the input data has been retrieved and manipulated, the normalized inputs are fed into the associated neural network engine. The resulting output of the calculation will be the gross forecast value.
Rollup Forecast

The process by which a meter group's gross forecast values are assigned based on the rollup of actual historical gas load values from the group's children.

If this option is selected the Rollups button is enabled and you can view the historical value rollup rule for the meter or meter group. See Viewing Historical Value Rollup Rules for more information.

Assignment from Parent

The process by which a meter or meter group can be assigned a forecast value from its parent instead of directly forecasting its own value. In some situations it might be more useful to perform forecasting at a group level, and then assign forecast values down to child meters, a process known as energy load assignment.

If this option is selected the Assignment button is enabled and you can configure an assignment rule for the meter or meter group. See Configuring Assignment From Parent for more information.

Aggregation from Children

If a meter group is not directly forecasting its own forecast value, it can be assigned a value based on an aggregation of the forecasts of its child meters.

If this option is selected the Aggregation button is enabled and you can configure an aggregation rule for the meter or meter group. See Configuring Aggregation From Children for more information.

Expression Redistribution

The process by which a single meter or meter group can retrieve its gross forecast value from another meter or meter group’s net forecast value.

If this option is selected the Redistribution button is enabled and you can configure an expression redistribution rule for the meter or meter group. See Configuring Expression Redistribution for more information.
Balancing Redistribution

The process by which you can conditionally distribute a calculated forecast value for a group of meters based on whether the difference of two summed sets of meters is positive or negative.

If this option is selected the Redistribution button is enabled with a drop-down arrow and you must select one of three options to configure a balancing redistribution rule for the meter or meter group:

  • Create New Rule — Select to open the Edit Balancing Redistribution Rule dialog box for the target meter to create a new balancing redistribution rule.
  • Edit Meter's Rules — Select to open the Redistribution Rules dialog box that displays only the rules of which this meter is a forecast recipient. Select a balancing redistribution rule to modify and click Edit Rule….
  • Add to Existing Rules — Select to open the Redistribution Rules dialog box for the target meter, from where you can add this meter to existing balancing redistribution rules.

See Configuring Balancing Redistribution for more information.
Assignment Summary

Specifies a summary of the configured Assignment Type rule(s) for the current meter or meter group.
Value Adjustment

Specifies the energy load adjustment setting that has been defined for this meter or meter group. An adjustment will occur whenever a meter or meter group’s gross forecast value is set by one of the defined Value Assignment Types. If no value adjustments are currently active, the default action will be to copy the gross forecast to the net forecast value.

Click Adjustment to configure an Adjustment Rule for the current meter or meter group. See Configuring Value Adjustment for more information.

Hierarchy Memberships

Parameter Description
Hierarchy Description

Specifies a list of possible Hierarchy Descriptions that the selected meter or meter group is eligible to join. A target meter or meter group can be a member of up to ten distinct hierarchies.

Note: A (no parent) note will display to the right of the selected Hierarchy Description if it has no parent.

Each Hierarchy Description belongs to a Meter Hierarchy Parent facility attribute that stores the information from where a hierarchy can be built. The facility attribute used to store the Meter Hierarchy Parent value is configured on the System Settings page. See Configuring Facility Attribute Settings for more information.
Use for Assignment

Click to indicate that the selected Meter Hierarchy Parent is to be used for value assignment.

Note: An asterisk (*) will display to the left of the selected Hierarchy Parent to indicate from which parent this meter or meter groups is getting its value assignment.

The facility attribute used to store which Meter Hierarchy Parent will be used for Value Assignment is configured on the System Settings page. See Configuring Facility Attribute Settings for more information.

Hierarchy Parent

Parameter Description
Top-level Hierarchy Member

Click to indicate that this meter or meter group has no parent, that is, this facility is the top-level node (root) in the specified hierarchy.

Note: For each meter or meter group, you must indicate whether it is at the top of a hierarchy, or set a Parent in the next field. Hierarchies can have multiple top-level members.
Parent Target ID

Specifies the facility attribute that will store the resolved CygNet Facility Tag referenced by the external Group ID value provided in the import file or the configured CygNet Facility Tag entered manually for this meter or meter group. The specific Meter Hierarchy Parent facility attribute used to store this value is configured on the System Settings page. See Configuring Facility Attribute Settings.

Click … to select a valid Parent Target ID.

Click >> to see the Assignments settings for the Parent Target in read-only mode.
Clear

Click to remove the Parent Target ID for this meter or meter group.

Note: You cannot remove a Parent Target ID if the meter or meter group is used in its parent aggregation rule.

Hierarchy Children

Parameter Description
Hierarchy Children

Specifies a list of the child meters or meter groups that belong to the currently selected meter group. Any one child can be a member of more than one hierarchy. For each child the list displays the Target ID, Description, External ID, Assignment Type, and Assignment Hierarchy.
Add Child Click Add Child to select a Facility ID to add to the hierarchy.
Remove Child Clink Remove Child to remove a selected Facility ID from the hierarchy.
Assignments to Children

Click Assignments to Children to view the Assignment Rules for the children of the selected target parent. See Configuring Assignment From Parent for more information. If none of the children get forecast values from a parent this button is disabled.

Configuration Errors

If the parent and child relationships and their associated rules are misconfigured in any way, errors will be displayed at the bottom of the dialog box. A single error will display directly on the dialog box; if two or more configuration errors exist an All Errors button will display. Errors must be resolved before OK is enabled.
