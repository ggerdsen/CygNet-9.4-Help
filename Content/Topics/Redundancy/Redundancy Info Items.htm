Redundancy and Failover Info Items

There are several SVCMON UDCs that can be used to monitor redundancy and failover using the CygNet ServiceMon Administration utility. The system info items upon which these UDCs are based can be viewed for each service on the Service Generic page of the CygNet Service Information dialog box.

Also see Replication Info Items and Monitoring VHS Replication for a list of SVCMON UDCs that can be used to monitor replication.

Note: It is recommended that the SVCMON timer be set for no more than five seconds.

Info Item Description SVCMON UDC
FAILOVER_READINESS

Indicates the readiness of the service to start a failover. The values include:

  • Not ready — The service is Not Ready if
  • the REPL_SUMMARY_STATE Info Item is 'Invalid' or 'Unknown' and
  • the FAILOVER_STATUS Info Item is any value other than 'Normal'.
  • Ready — The service is Ready if
  • the REPL_SUMMARY_STATE Info Item is 'Valid' and
  • the FAILOVER_STATUS Info Item is 'Normal'.
  • Unknown — The failover readiness of the service is unknown.
 SVMFOREADY
FAILOVER_STATUS

Indicates the current failover status of the service. The values include:

  • Not redundant— For a non-redundant service.
  • Normal — Not currently doing a failover. Applies to all services.
  • Waiting — Waiting for the next step in the failover sequence. Applies to all services.
  • Freezing — In the process of stopping all processing. Applies to all services except RSM.
  • Frozen — All processing has stopped. Applies to all services except RSM.
  • Replicating — The service is catching up on replication. Applies to all services except RSM.
  • Processing — The master RSM service is doing work. Applies to the master RSM only.
  • Pre-failover — The RSM is waiting for all services to synchronize. Applies to RSMs only.
  • Starting — The service is starting a failover. Applies to RSMs only.
  • Failover — Switching services from standby to active. Applies to RSMs only.
  • Post-failover — Starting standby services and changing replication source of indirect services. Applies to RSMs only.
  • Aborting — Aborting failover. Applies to RSMs only.
  • Error — The service is in error. Applies to all services except RSM.
  • Monitoring — Not currently doing a failover, but monitoring the state of the RSM on the Primary Active Server. Applies to RSMs only.
  • Unknown — The failover status of the service is unknown.
 SVMFOSTATU
IN_REDUNDANT_SET

Indicates whether this service is part of a redundant relationship. The value equals the REDUNDANT keyword in the service configuration file.

 SVMINRDSET
