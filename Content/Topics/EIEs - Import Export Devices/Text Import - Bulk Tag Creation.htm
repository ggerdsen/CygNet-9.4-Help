Bulk Tag Creation

An import can be created to import current values, point configurations, facility attributes, or any combination of the three. By default, both current values and point configurations are imported.

If you are using the Text Import driver to import tag definitions into the system, the tags' current value service will be the UIS with which the DDS is associated. (A one-to-one relationship exists between a UIS and a DDS. There are parameters in the configuration file of these services that define this relationship.)

The tags' PNT will be that defined by the UIS. (A UIS is configured to retrieve its tag information from a specific PNT. This relationship is defined using parameters in the UIS configuration file.)

Alternate Current Value Service

You can specify an alternate current value service by using the Alternate Current Value Service browse button on the Text Import Properties Advanced page. This alternate service can be a UIS, HSS, OPCIS, or SVCMON. The tag definitions will be stored in the PNT associated with that specified current value service instead. You can also replace a history value when importing to an alternate CVS.

When importing to an alternate CVS, CygNet uses a setpoint to set the values. There are some special considerations to keep in mind.

Considerations

  1. If the import creates/updates PNT records, it is possible that the CVS does not have the most up-to-date point information when it gets the new value.
  2. There are no special considerations for importing historical data. The CVS shows the most recently updated value, not the one with the latest timestamp.
  3. The template tags are read from the PNT associated with the TextImport UIS, not the alternate CVS.
