Filtering Data

A filter can be applied to limit the data imported into the system. There are two components that define the filter:

  1. The Import Filter component on the Column Mapping page, and
  2. The Filter Value field on the Advanced page.

Select Import Filter under Text Import Attributes for the Component on the Column Mapping page. The Import Filter component specifies which column in the source file to filter on. You can assign the Import Filter component to any column in the source file, regardless of whether the column is mapped to another component or not.

Hint: To ignore all tags for which the value in a column is blank, set the Filter Value to "?*", meaning it must include one or more characters.

The Filter Value field specifies the match for the filter. Only data for tags that match the filter will be imported into the system. Wildcards ("*" and "?") can be used.

Example

The source file below contains data for 48 tags: each row represents a different meter; there are four values per row.

To import data for only Meter ID METER1, set the Filter Value to "METER1" and the Import Filter component column to "%col1%" (column 1 contains the Meter ID).

Click Test on the Device page to see the anticipated results: that the data for Meter1 will be imported into the system (16 tags), while all other data is filtered out (32 tags).
