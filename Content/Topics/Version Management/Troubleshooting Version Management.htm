Troubleshooting Version Management

The following tips might be helpful in solving issues that arise in version management.

I’ve updated my RSP timestamp but users still aren’t getting the new EIE editor.

You must also update the timestamp of any corresponding .rsq files and any referenced .rsp files. For editors, this generally includes CxDdsEditors.rsq and the EIE’s .rsp file (for example, CxDdsModbusEditors.rsp).

How can I tell if version management is occurring?

If the Version Manager is running, you will see a Transfer Progress dialog box. If you don’t see the dialog box, version management isn’t necessary or has already run according to your version management settings defined in the CygNet Client Installer. You can trigger version management by deleting the application’s .rsp file on your local drive.

Can I create shortcuts on my desktop or other locations?

Yes. One way is to copy the shortcut from the Start menu. This will ensure that the shortcut contains the version management information. Client Installer also provides the option to create a desktop shortcut when installing a new application. This shortcut will also use version management. We do not recommend creating shortcuts that do not include version management.

What are .rsp files and why do I need to "touch" them?

An .rsp file lists all of the files associated with an application. When a new version of an application file is uploaded to a BSS, the corresponding .rsp file needs to have its timestamp updated to trigger version management. Rather than forcing you to open and resave the file to update the timestamp, we have provided a utility to update the timestamp. This utility is TouchBSS.exe.

How do I upload a new version of a file into the BSS?
  1. Using CygNet Explorer, select the BSS.
  2. Right-click on the file to be replaced and select Replace File.
  3. Browse to the directory that contains the newer file, select the file, and click Replace.

Note: Don’t forget to update the timestamp of the corresponding .rsp/.rsq files so that version management is initiated for the new file.

-OR-

  1. Using BSS Explorer, select a folder and click Upload or right-click and select Upload Files.
  2. Select the desired folder from the Select Blob Folder dialog box. Type a new folder name if necessary.
  3. Browse to the file(s) and click Upload to upload to the selected folder.

Note: You can drag one or more files from the desktop or other file system location into a folder.

Why doesn’t my computer have the latest version of software?

Either the BSS does not have the latest file, the .rsp/.rsq file does not contain the correct list of file components, the .rsp/.rsq file has not been touched, or the shortcut does not start the Version Manager.

How can I circumvent version management?

You can click on the application executable or set your CygNet Client Installer advanced options setting to "None" or "Prompt." If you have it set to Prompt, click "No" when it asks if you want to localize files.
