Defining Audit Levels

Auditing is configured on a per service basis. Some services have multiple auditing keywords to allow record changes of a certain type to be more detailed than others. This allows more customization of how a service’s records are audited. For example the DDS has auditing keywords specific to data groups and UIS commands; each has its own auditing level.

Using CygNet Explorer to Change Service Audit Levels

From CygNet Explorer, right-click on any service in the left-hand service hierarchy pane and select Service Details. Click Options on the lower left, and choose Audit Level Settings.

Modify any keyword as necessary and click to Send new Audit Levels to Service to temporarily change audit levels. Note changes are temporary and will be lost when the service is restarted.

See Service Audit Levels for more information about the keywords for each service.

See the SVCINFO Security Event for the applicable CygNet Service in the Security Reference topics for more information about security access levels for changing audit levels.
