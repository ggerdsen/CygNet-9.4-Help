System: Precalculate Calibration Data

Note: This command operates in conjunction with CygNet Dispatch. For more information about the CygNet Dispatch product, contact your Account Manager or CygNet Sales.

Precalculate Calibration Data (JOBPREC via script) is a system-level command that triggers the service to precalculate the effective impact of data changes, as a result of a CygNet Dispatch Calibration job for a specified group or device Node. Precalculation results in a status of "Missing Data" when the set of historical data for the record span is incomplete, or "Pending Approval" when the set of historical data for the record span is complete. The record span is defined as the time between the Effective Date (the start date from which data recalculations will be applied) and the Completed Date (the date on which the job record was saved as complete; also denotes the end date to which data recalculations will be applied). Historical data is not recalculated at this point.

Note: Although this precalculation command can be run in either operational mode, REPOSITORY or FULL, to determine the potential impact of value changes provided via CygNet Dispatch, the recalculation functionality needed to apply those changes is only available in CygNet Measurement systems licensed for FULL mode.

You must have proper security authorization to execute this command. See FMS Security (CMDSYS and JOB security events) for information about configuring security access for FMS commands.

Managing Precalculation Commands

Prior to performing Precalculate Calibration Data commands, results from CygNet Dispatch must already be loaded into the FMS service. See CygNet Dispatch for more information about the field calibration data stage of the process.

When the Precalculate Calibration Data command is run, preliminary calculations are performed by the FMS service to determine the potential volume changes resulting from the calibration data changes, and those results are made available for action in the Jobs control.

After the Precalculate Calibration Data command is run, a Status of "Missing Data" or "Pending Approval" is displayed in the Jobs control.

  • When a status of "Missing Data" is present, import, edit, or poll for data until all data is present and a status of "Pending Approval" is achieved.
  • Once a status of "Pending Approval" is achieved, additional data actions (polls, edits, purges, closed periods, etc.) will automatically update the precalculation result values.

Note: Records are not recalculated until approved via the Jobs control, and recalculations are then applied only to the record types specified at the time of processing. See Using the Jobs Control for more information about the recalculation stage of the process.

Command Definition Properties

The following command definition properties apply to Precalculate Calibration Data commands, when scheduling, creating, or editing commands via the Master Scheduling Service (MSS) in CygNet Explorer.

Property Description

Priority

Select a processing queue Priority for the command from the drop-down menu.

Options from least to greatest priority are as follows.

  • Low level
  • Medium level
  • High level
  • User initiated transaction
  • Admin initiated transaction

Default value is Low level processing queue priority.
Logging

Select a Logging level for the command results from the drop-down menu.

Options are as follows.

  • Low - displays errors
  • Medium - displays errors and warnings
  • High - displays errors, warnings, and status

Default value is Low level logging when scheduling a command via the MSS service in CygNet Explorer.

Default value is High level logging when creating or running an ad-hoc command in FMS Explorer or via the FMS service in CygNet Explorer.

Command Parameters

No command parameters appear in the dialog box for this command. Use the Device descriptor(s) section of the dialog box to define applicable group or device Nodes to include in the Precalculate Calibration Data action.
