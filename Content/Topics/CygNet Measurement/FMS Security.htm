FMS Security

Security in CygNet Measurement is enforced by the CygNet Access Control Service (ACS). See Security for more information about CygNet Software security.

Security Keywords

The CygNet Measurement security relationship is defined in the "Security Keywords" section of the FMS service configuration file (CygNet\Services\FMS\fms.cfg on your host server).

The user ID of the service is defined using the security keyword ACS_ID. The security application name of the FMS service is defined using the security keyword ACS_APPLICATION. The main security event name is defined using the security keyword MAIN_SEC_EVENT.

See Service Configuration Files and FMS Configuration File Keywords in the Services section for more information.

Security Applications and Events

Security is set on an Application and Event basis, as it is for other CygNet services.

  • Application - The security applications used in FMS can be either service-level (FMS service) or component-level (FMS Node). See FMS Security Applications for more information.
  • Event - The security events used in FMS can be standard or custom, and are listed in tables including the FMS tasks and the associated authorization levels required to be granted permission. See FMS Security Events for more information.
