Configuring FMS Redundancy

Note: CygNet Redundancy operates on top of the CygNet replication model and is supported by underlying replication functionality for all services, including FMS when so configured. Certain FMS replication and data synchronization keywords must be disabled for an FMS service to be included in a redundant environment for data-center failover.

The Flow Measurement Service (FMS) can be included in a redundant set of services in order to support failover. However at this time the FMS only supports data-center failover, due to the structure of the FMS, and it does not support local redundancy. Local redundancy can be achieved with Microsoft SQL server replication and/or clustering.

A redundant RSM will not start an FMS on a domain defined as in the local standby role. If you are configuring a local failover with a site that includes an FMS, the FMS can be included, but the local standby FMS service will not start. Use the CygNet Redundancy Editor to configure the role played by a domain in a redundant environment.

See CygNet Redundancy for more information.

Configuring FMS Data-Center Redundancy

To enable data-center redundancy for an FMS, values must be set using the appropriate service configuration file keywords. This can be enabled in a single action using the Configuration File Manager utility. See CygNet Config File Manager for more information.

If the FMS is configured to be redundant, the following requirements apply for values for keywords in the FMS service configuration file:

Keyword

Required

Comment
REDUNDANT Yes

Must be set to TRUE to include the FMS in a redundant environment for data-center failover.
DS_MODE No

Do not configure, leave commented.

Note: If a value is present for this keyword, the FMS service will fail to start and an error will be logged to the service log file.
DS_ROLES No

Do not configure, leave commented.

Note: If a value is present for this keyword, the FMS service will fail to start and an error will be logged to the service log file.
DS_REPL_MASTER No

Do not configure, leave commented.

Note: If a value is present for this keyword, the FMS service will fail to start and an error will be logged to the service log file.
DS_REPL_SLAVE No

Do not configure, leave commented.

Note: If a value is present for this keyword, the FMS service will fail to start and an error will be logged to the service log file.
DS_CHECK_INTERVAL Yes Required when the FMS is included in a redundant environment for data-center failover.
DS_DELAY_MAX No Although not required, it is highly recommended that this keyword is configured.
DS_QUICK_DB_COMPARE No Does not apply to FMS Redundancy. Leave commented.
DS_BLACKOUT_START No Does not apply to FMS Redundancy. Leave commented.
DS_BLACKOUT_END No Does not apply to FMS Redundancy. Leave commented.

Once redundancy is configured for FMS, you can execute a manual data-center failover that includes an FMS server, and view the failover in the Redundancy Dashboard.

Monitoring FMS Redundancy and Failover

The FMS also supports and will return values for the following related info items:

Service Generic Info Items

  • FAILOVER_READINESS
  • FAILOVER_STATUS
  • IN_REDUNDANT_SET

Replication Info Items

  • FMS_DS_CHK_INTERVAL
  • REPL_DELAY
  • REPL_DELAY_MAX
  • REPL_INT_VAL_STATE
  • REPL_STATUS
  • REPL_SUMMARY_STATE
