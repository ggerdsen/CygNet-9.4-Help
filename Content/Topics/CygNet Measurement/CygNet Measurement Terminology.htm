CygNet Measurement Terminology

The following table contains definitions of specific terminology used in CygNet Measurement.

Term Definition

Alarm data item

(eFMS_ALARM_*)

A value associated with the alarm states of an EFM device, providing actionable information about the state of the data, based upon user-configured guidelines.

Example

DP alarm, battery alarm, cutoff alarm, alarm state values, etc.

See EFM Data items and FMS Alarm Items in the Devices section for more information.

Alarm data

A generic term used to describe multiple alarm records.

Alarm record

 A set of historical alarm data item values that notes and records all exceptions to certain process variable values contained within a periodic metering or periodic quality record.

See Using the Alarm Control for more information.

Alternate GQ/GA source

A Node designated as the source Node to use for gas quality or gas analysis data, when live GQ data is not available for a selected device Node.

See Creating Gas Meter Nodes for more information.

Batch data item

(eFMS_BATCH_*)

A value associated with a liquid product "batch" describing the product type, quantities, qualities, or ownership details.

Example

Batch type, density, EVP, pulses, etc.

See EFM Data items and FMS Batch Items in the Devices section for more information.

Batch data

A generic term used to describe multiple batch records.

Batch record

A set of historical batch data item values in a preset format that supports the determination of liquid quantities for a specific batch transaction.

Close period

An action to finalize "best" data records for a specific date range, indicating that any issues have been resolved. Any change beyond this time requires a PPA.

See Closing Data in the Managing CygNet Measurement Data topic for more information.

Configuration data item

(eFMS_CFG_*)

A value that affects the calculation, is not associated with a property or state of the flowing gas or liquid, and typically does not frequently change.

Example

Orifice diameter, calculation method, atmospheric pressure, etc.

See EFM Data items and FMS Configuration Items in the Devices section for more information.

Configuration data

A generic term used to describe multiple configuration records.

Configuration record

 A set of configuration data item values that contains and identifies all selected flow parameters used in the generation of a periodic metering record.

Contract day

A 24-hour definition of a "day" defined by an offset (in hours) from the 00:00:00 (midnight) Device time by the "contract hour" setting.

See Using Contract Hour in CygNet Measurement for more information.

Contract hour

A device configuration item, defined as an offset (in hours) from the 00:00:00 (midnight) Device time, indicating the hour of the day at which the 24-hour "contract day" begins.

See Using Contract Hour in CygNet Measurement for more information.

Contract time

A time format in which data can be viewed, defined by an offset (in hours) from the 00:00:00 (midnight) Device time. It is a relative time, determined by the "contract hour" offset at which the "contract day" begins.

See Using Contract Hour in CygNet Measurement for more information.

CygNet Dispatch

An optional product that can integrate field technician activities with your CygNet Measurement system. CygNet Dispatch includes job scheduling, tracking, and reporting functionality for calibration and inspection tasks.

For more information about the CygNet Dispatch product, contact your Account Manager or CygNet Sales.

Estimation

A process performed by CygNet to find missing, late, or poor quality data and provide higher quality estimated values. The processing is applied to station level data. Data is estimated in compliance with specified estimation rules, which are user-configurable.

See Estimation in the CygNet Measurement Concepts topic for more information.

Event data item

(eFMS_DEVICE_EVENT_*

or eFMS_CUSTOM_EVENT_*)

A value that is associated with a measurement system event, causing a change to an element used as part of a calculation or record.

Example

Device ID, date, time, custom-defined user event(s), etc.

See EFM Data items and FMS Event Items in the Devices section for more information.

Event data

A generic term used to describe multiple event records.

Event record

A set of event data item values that notes and records all exceptions and changes to the flow parameters contained within a configuration record that occur and have an impact on a periodic metering record.

Exception

A notification produced by the system to provide information that data has failed a validation rule.

See Exceptions in the CygNet Measurement Concepts topic for more information.

Gas analysis data

A generic term used to describe multiple imported gas analysis records.

Gas analysis record

A set of imported gas analysis configuration data item values that affects the calculation, is not associated with a property or state of the flowing gas or liquid, and does not frequently change.

Gas device

A single gas measurement device, such as a gas meter or chromatograph, that collects electronic flow measurement data for gas products.
Gas quality data A generic term used to describe multiple gas quality records.

Gas quality record

A set of gas quality configuration data item values that affects the calculation, is not associated with a property or state of the flowing gas or liquid, and does not frequently change.

History data item

(eFMS_PV_*)

A value associated with the flow or state of an EFM device, providing a periodic metering or periodic quality data item value, that is input into the service computer for the purpose of being part of a calculation.

Example

Differential pressure, temperature, volume, % methane, etc.

See EFM Data items and FMS History Items in the Devices section for more information.

History data

A generic term used to describe multiple history records, such as periodic metering and periodic quality records.

History record

A set of history data item values, calculated values, and information in a preset format that supports the determination of a quantity over a specific accounting period.

Liquid device

A single liquid measurement device (with periodic, batch, and/or ticket history) that collects electronic flow measurement data for liquid products.

Node

An organizational unit in CygNet Measurement, which can be configured to represent a device, station, or group of various configurations.

See Nodes for more information.

Normalization view

A definition that standardizes the format of varying timestamps, base conditions, unit sets, etc. to allow streamlined processing and viewing of station-level data.

See Normalization Views in the CygNet Measurement Concepts topic for more information.

Normalized record

A record normalized from non-hourly records (sub-hourly records, and records not aligned with the top of an hour) to strict hourly records in order to simplify downstream business processes and analysis. Also known as an Hour Normalized Record (HNR).

See Normalization of Data in the CygNet Measurement Concepts topic for more information.

Periodic metering history

A generic term used to describe multiple periodic metering records. Also known as PM History, or QTR. (API 21.1: QTR - Quantity Transaction Record)

Periodic metering record

A set of historical data item values, calculated values, and information in a preset format that supports the determination of a quantity over a specific accounting period.

Periodic quality history

A generic term used to describe multiple periodic quality records. Also known as PQ History.

Periodic quality record

A set of historical data item values, calculated values, and information in a preset format that supports the determination of the periodic quality over a specific accounting period.

Prior period adjustment

(PPA)

An adjustment made to a value in a previously closed period, that may be required due to an edit or data change arriving in the system after the corresponding period has been closed.

See Prior Period Adjustments in the CygNet Measurement Concepts topic for more information.

Process variable

(PV)

A History data item used to convey a calculation input value.

Product data item

(eFMS_LIQUID_PRODUCT_*)

A value associated with liquid product transactions, describing the product type, density calculation, or conditions.

Example

Density calculation used, product type, compressibility factor, prove date, etc.

See EFM Data items and FMS Product Items in the Devices section for more information.

Product data

A generic term used to describe multiple liquid product records.

Product record

A set of liquid product data item values in a preset format that describes liquid product characteristics within a specific time period.

Quantity transaction record

(QTR)

See Periodic Metering History in this table.

Session

A definition of system conditions, consisting of Node, time window, and time format information, to which many controls can optionally subscribe simultaneously.

See Sessions for more information.

Session Node

A Node selected to be the Node setting to which many controls can subscribe simultaneously via the Use Session Node option.

See Session Node for more information.

Session options

A combination of time window and time format settings to which many controls can subscribe simultaneously via the Use Session options option.

See Session Options for more information.

Smart group

An automatically-generated group association based on user-specified conditions and rules, initiated by a Generate Smart Groups command.

See Generate Smart Groups for more information.

Station

A group of measurement devices, that reports electronic flow measurement data.

Stations can be one of the following types.

  • Virtual station - contains other stations or liquid devices
  • Physical station - contains one or more gas devices

See Device Data and Station Data in the CygNet Measurement Concepts topic for more information.

Ticket data item

(eFMS_TICKET_*)

A value associated with a field ticket transaction, describing the product type, quantities, qualities, or ownership details.

Example

Ticket type, shipper, viscosity, observed volume, etc.

See EFM Data items and FMS Ticket Items in the Devices section for more information.

Ticket data

A generic term used to describe multiple ticket records.

See Ticket Reports or Exporting Ticket Data for more information.

Ticket record

A set of historical ticket data item values in a preset format that supports the determination of liquid quantities for a specific ticket transaction.

Time format

A description of the time format in which your data can be viewed or processed. Options are Device time (D), Contract time (C), and Local time (L).

See Using the Session Menu for more information.

Validation

An analysis process performed by CygNet to ensure that measurement data is as accurate as possible, and to help identify data of insufficient quality. Data is validated to check compliance with specified validation rules, which are user-configurable.

See Validation in the CygNet Measurement Concepts topic for more information.

Workspace

An FMS Explorer configuration local to each client, capable of automatically saving current settings when the application is closed, and reloading the settings when opened.

See Using the File Menu for more information.
