CygNet Measurement Software Components

CygNet Measurement consists of the following software components, and also utilizes additional components depending on system configuration choices.

CygNet Measurement Components

FMS Internal Database

Note: See Using the FMS Internal Database for information about database selection.

The FMS Internal Database is provided by CygNet Software to manage FMS data in appropriate low-use environments, without using SQL queries. If using a SQL Server Database, this component is not necessary.

For most CygNet Measurement applications, the FMS Internal Database is not used, and a Microsoft SQL Server database is used instead.

FMS Service

The FMS Service is the CygNet Software service that is the interface between the database and the SCADA system.

See FMS Service for more information.

FMS Explorer

The FMS Explorer is the CygNet Software client application for viewing, editing, recalculating, approving, reporting, and exporting gas measurement data.

See FMS Explorer for more information.

FMS Toolbox

The FMS Toolbox is the CygNet Software utility for migrating and importing files into the CygNet Measurement system.

See FMS Toolbox Utility for more information.

FMS Internal Database Manager

The FMS Internal Database Manager is the CygNet Software utility for managing the (optional) FMS Internal Database using SQL queries. If using the SQL Server Database, this component is not necessary.

See FMS Internal Database Manager Utility for more information.

CygNet Explorer

Although not specifically a CygNet Measurement application, CygNet Explorer is required to configure the CygNet Measurement application, security tasks, data archiving, and scheduled tasks such as data retrieval. It can also be used to edit Node properties and execute FMS quick commands.

See CygNet Explorer for more information.

CygNet Dispatch (Optional)

Although not required, CygNet Dispatch is available separately to add job scheduling, tracking, and reporting functionality for calibration and inspection tasks to your CygNet Measurement installation. It can be used by technicians in the field and integrated via the FMS service.

See CygNet Dispatch for more information.

System Requirements

Additionally, CygNet Measurement relies on the following external components.

Note: See the CygNet System Requirements document for more information about these and other software requirements.

Microsoft SQL Server Database

Note: See Using the FMS Internal Database for information about database selection.

If using the FMS internal database, this component is not necessary.

For most CygNet Measurement applications, the Microsoft SQL Server database is used and must be configured to work with CygNet Measurement.

  • Microsoft SQL Server must meet the requirements listed in the CygNet System Requirements document.

Installation of the Microsoft SQL Server is outside the scope of this document. Refer to Microsoft SQL Server documentation for more information.

Microsoft SQL Server Native Client

If running Microsoft SQL Server and the FMS service on the same computer (not generally recommended), this component is not necessary.

For most CygNet Measurement applications using the Microsoft SQL Server database, the Microsoft SQL Server and the FMS service will be run on different physical computers (recommended), therefore the Microsoft SQL Server Native Client must be installed on the computer hosting your FMS service.

  • Microsoft SQL Server Native Client must meet the requirements listed in the CygNet System Requirements document.

Installation of the Microsoft SQL Server Native Client is outside the scope of this document. Refer to Microsoft SQL Server Native Client documentation for more information.

Oracle Database Client

If running CygNet Measurement in FULL mode, or running in REPOSITORY mode but not using the optional licensed feature set, this component is not necessary.

For CygNet Measurement applications running in REPOSITORY mode, and using the optional licensed feature set requiring access to the bi-directional database interface, the Oracle Database Client is required.

  • Oracle Database Client must meet the requirements listed in the CygNet System Requirements document.

Installation of the Oracle Database Client is outside the scope of this document. Refer to Oracle Database Client documentation for more information.
