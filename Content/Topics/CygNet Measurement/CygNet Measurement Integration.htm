Integrating CygNet Measurement with SCADA, Dispatch, and Third-Party Systems

CygNet Measurement is designed to integrate with SCADA, Dispatch, and third-party systems through its Flow Measurement Service (FMS).

Integration with SCADA

The FMS service can receive measurement information, by station, from the SCADA system. Gas loss information is sent to FMS using the device facility tag, since the SCADA system does not know about device names, and the device facility tag is treated like any other data coming in from an external source. In this way, data such as line pack changes and known gas loss information can be sent from SCADA to FMS.

See FMS Commands and Running File Import Commands for information about importing various device data files into FMS.

Integration with CygNet Dispatch

The FMS service can receive and report on field technician activities by integrating CygNet Dispatch into your CygNet Measurement system. This optionally available feature set includes job scheduling, tracking, and reporting functionality for calibration and inspection tasks, and possible adjustments based on changes affecting data calculations.

See CygNet Dispatch for more information.

Integration with Third-Party Systems

CygNet Measurement can share data with third-party systems such as production accounting systems or (optionally) Flow-Cal or PGAS to provide desired business functionality.

Production Accounting Systems

Export closed daily and monthly volume and energy values, approved PPA records for a device, group of devices, or rollups, and other FMS controls data, from CygNet Measurement to production accounting systems via .csv file exports or other means. This can be achieved manually or via scheduled actions. CygNet Measurement tracks data that has been exported from the system.

See the following topics for information about data exporting options from FMS.

  • Managing CygNet Measurement Data - overview of available data exporting functionality
  • FMS Commands - list of available FMS export commands
  • Using the Controls Menu - access to reports, administrative (Close, PPA, Balance), and data controls for exports via FMS Explorer

Flow-Cal Systems

Integrate gas quality and measurement information between CygNet Measurement and Flow-Cal via installation of optionally available FMS commands.

See the following topics for information about the optionally available functionality available for integration with Flow-Cal systems.

  • Flow-Cal Request Missing Data - scan a Flow-Cal database for missing data requests
  • Flow-Cal Send Gas Quality - scan a Flow-Cal database for gas quality requests to send to field devices
  • Flow-Cal Export Data CFX - export raw periodic and/or batch data as a .cfx file
  • Flow-Cal Export Ticket Data TFX - export raw ticket data as a .tfx file
  • Flow-Cal Export Transaction Queue - export raw gas periodic records to a Flow-Cal database transaction queue

See Installing CygNet Measurement for information about providing support for these optional commands.

PGAS Systems

Integrate measurement data between CygNet Measurement and Quorum PGAS via installation of an optionally available FMS command.

See the following topic for information about the optionally available functionality available for export to PGAS systems.

  • PGAS Export Data XML - export raw measurement data as a PGAS XML file
