Managing Node Commands

Use the Manage button to access options to Add, Edit, or Delete FMS Node commands through the FMS Explorer Command menu. Configured command definitions are saved as part of the workspace when you choose to Save your current workspace.

To manage commands, open the Commands menu in FMS Explorer, and then click Manage to access the FMS Command Definitions dialog box.

Currently configured Node commands, including the Command Name and the Command Type, appear in the dialog box and are listed in alphabetical order.

Creating a New FMS Command Definition

To create a new command definition, click Add to access the Create New FMS Command Definition dialog box.

See the following command configuration topics for specific configuration information and corresponding parameters for each command.

  • Close Period
  • Request Current Configuration
  • Request Data
  • Request Missing Data
  • Request New Data
  • Request Product Data
  • Send Gas Quality
  • Validate Data
  • Export: Flow-Cal Data CFX (if installed)*
  • Export: Flow-Cal Ticket Data TFX (if installed)*
  • Export: Flow-Cal Transaction Queue (if installed)*
  • Export: PGAS Data XML (if installed)*

*Requirements for Optional Commands

Flow-Cal and PGAS data export commands are provided as optional features. To enable the export of raw data to Flow-Cal via database/CFX/TFX files or PGAS via XML files, CygNet Measurement provides optional files that can be installed on your system. These include CDF files that contain the data item mappings and actions necessary to execute the raw data export commands for multiple file types.

For the optional Flow-Cal commands, two conditions must be met.

  • The CxFmsFlowCalExport.dll file must be included and registered in the Support folder on your host server.
  • For each export type, the corresponding command definition file (CDF) must be included in the CygNet\Services\FMS\CommandDefs folder.

See Installing CygNet Measurement for more information about installation and registration of the optional support files.

For the optional PGAS command, the following condition must be met.

  • For a PGAS export, the corresponding command definition file (CDF) must be included in the CygNet\Services\FMS\CommandDefs folder.

Editing an FMS Command Definition

Current command definitions appear in the FMS Command Definitions dialog box, where they are available for editing.

To Edit an FMS Command Definition

  1. In the command definitions list, do one of the following to enable the option to edit a command.
    • Double-click on the command name.
    • Click to select the command name and then click Edit.

      The Edit FMS Command Definition dialog box will appear.

  1. Make edits to the fields required, as for Creating a New FMS Command Definition.
  2. Click Save Changes when you are done editing the FMS command.

Deleting an FMS Command Definition

Current command definitions appear in the FMS Command Definition dialog box, and can be deleted if desired.

To Delete an FMS Command Definition

  1. In the command definitions list, click to select the Command Name you wish to delete. This will enable the option to delete the command.
  2. Click Delete to remove the command definition.

    Note: Be certain of your deletion, as there is no confirmation prompt.

See Running Node Commands for more information about executing FMS Node commands.
