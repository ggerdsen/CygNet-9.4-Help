System: Estimate Station Data

Note: This functionality is only available in systems licensed for FULL mode.

Estimate Station Data (STATIONEST via script) is a system-level command that triggers the service to estimate station level data, in certain defined conditions and using specified methods, to achieve better data values. These conditions and methods are defined by Estimation Engines and Rules.

You must have proper security authorization to execute this command. See FMS Security (and CMDSYS security event) for information about configuring security access for FMS commands.

See Managing Estimation Engines and Running Estimation Commands for more information about using this command via FMS Explorer.

See Scheduling FMS Command Tasks in the MSS for more information about scheduling commands.

See FMS ActiveX Controls and CxHmiFms for more information about using commands via FMS ActiveX controls on CygNet Studio or CygNet Vision screens.

See CxFms for more information about scripting commands.

Managing Estimate Station Data Commands

Prior to running Estimate Station Data commands, verify or configure the estimation engines that define the estimating process. Estimation engines are configured by users with proper security access, via the Estimation button on the FMS Explorer Admin menu. See Managing Estimation Engines for more information.

Command Definition Properties

The following command definition properties apply to Estimate Station Data commands, when scheduling, creating, or editing commands via CygNet Explorer.

Property Description
FMS service Select a Site.Service for the command from the available FMS service list in the drop-down menu.
Command

Select the desired Command name from the drop-down menu.

See FMS Commands for more information about available command types.
Priority

Select a processing queue Priority for the command from the drop-down menu.

Options from least to greatest priority are as follows.

  • Low level
  • Medium level
  • High level
  • User initiated transaction
  • Admin initiated transaction

Default value is Low level processing queue priority.
Logging

Select a Logging level for the command results from the drop-down menu.

Options are as follows.

  • Low - displays errors
  • Medium - displays errors and warnings
  • High - displays errors, warnings, and status

Default value is Low level logging when scheduling a command via the MSS service in CygNet Explorer.

Default value is High level logging when creating or running an ad-hoc command in FMS Explorer or via the FMS service in CygNet Explorer.

Command Parameters

The actual estimation engines used in implementing the command do not appear in this dialog box, as they are too complex to be shown in this format. Estimation engines and rules, and their associated Nodes, can be viewed and configured via the Estimation button on the FMS Explorer Admin menu by users with proper security access. See Managing Estimation Engines for more information.
