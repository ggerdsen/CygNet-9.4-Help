Configuring Job Schedules

Note: The Configure Jobs command operates in conjunction with CygNet Dispatch. For more information about the CygNet Dispatch product, contact your Account Manager or CygNet Sales.

Use the Job Schedule Configuration page of the Extended Command Options dialog box to configure details for job schedules that can be utilized by the Schedule Jobs command.

Note: The definition of the Schedule Jobs command resides in FMS, but you can also use CygNet Explorer (MSS service) to schedule the command. Scheduling a Schedule Jobs command in the MSS will ensure that new jobs are added regularly. See Scheduling FMS Command Tasks in the MSS for more information.

You must have proper security authorization to configure Job Schedules. See FMS Security Events (and JOB security event) for information about configuring security access to extended command options.

See Running Node Commands for more information about running the configured command via FMS Explorer.

See Schedule Jobs for more information about additional options for using the command.

Job Schedules

Job schedules are utilized by Schedule Jobs commands to auto-generate jobs for CygNet Dispatch. Configure job schedules from the Commands menu, using the Extended Options button to access the job schedule configuration page. Each job schedule is based on user-defined parameters and an associated job frequency (fixed or variable). When Schedule Jobs commands are run, one or all of the job schedules are utilized, as indicated in the command parameters, to create and schedule jobs that meet the defined criteria. The jobs become available in CygNet Dispatch when it is synced with the service. See Schedule Jobs for more information.

Create or delete job schedules in the top section of the configuration pane, and define job schedule frequency for a selected schedule in the bottom section. Each schedule contains user-supplied properties (Job Schedule Name, Node, Schedule Window, etc.) that define the scheduled jobs that will be created when a Schedule Jobs command is run using that schedule. For each schedule listed, define its job frequency in the bottom section of the configuration pane to specify when each job will be required, how long the job will be available, and what specific recurrence pattern (Fixed frequency) or data values (Variable frequency) to use when scheduling jobs for CygNet Dispatch.

Possible values for job schedule properties are as follows.

Property Description
Job Schedule Name

Type in a Job Schedule Name to identify each new schedule.

Allowable characters (maximum of 20) are A-Z, a-z, 0-9, . (period), #, $, &, (, ), +, -, \, /, :, <, >, =, @, [, ], ^, {, }, _ (underscore), !, ~ (tilde). Spaces are acceptable, when they are not leading or trailing.

This property is required.

Node

Type in a Node name, or click ... to access the Select Node chooser to select a Node to create jobs for using this schedule.

This property is required.
Job Type

Select the Job Type for the jobs created by this schedule, from the drop-down menu.

Options are as follows.

  • Auto-calibration
  • Calibration
  • Gas Sample
  • [User-supplied value options]

Note: Job types are configured in CygNet Dispatch or in FMS Explorer via this interface.

This property is required.
Report Command Name

Select an existing Report Command Name from the available choices in the drop-down menu.

The job report template specifies the associated job report command name that is run to publish job information associated with the selected Node and time range. See Job Report Commands or Late Job Report Commands for more information.

This property is required.
Schedule Window

Click to access the Select a Schedule Window dialog box to specify the length of time you want to schedule jobs ahead when the Schedule Jobs command is run using this schedule. The schedule time window begins when the command is executed.

Specify the time window value in weeks, months, or years. For recurrences with units of month(s) or year(s), the cycle begins on the day of the month defined in the job schedule frequency as the "Required by" day.

Note:Coordinate the Schedule Window value you enter here with the recurrence value of the Schedule Jobs command configured in the MSS, to ensure constancy of scheduled jobs over time and to avoid leaving time periods that are unavailable for scheduling jobs.

This property is required.

Adding or Editing a Job Schedule

Use the following steps to create and configure Job Schedules for FMS Schedule Jobs commands.

To Configure a Job Schedule

  1. Open the Command menu in FMS Explorer, and click Extended Options to access the Extended Command Options dialog box.
  2. Click Job Schedule Configuration to access the job schedule configuration page.
  3. In the top pane, configure the schedule properties. Click Add to configure a new schedule, or select an existing Job Schedule Name to edit an existing schedule, and enter or edit the following properties.
    • Job Schedule Name
    • Node
    • Job Type
    • Report Command Name
    • Schedule Window
  4. In the bottom pane, configure the Job Schedule Frequency parameters. Select either a Fixed or Variable frequency to schedule recurring jobs.
  • Click Fixed to access the property fields required to configure a simple, fixed frequency for the scheduled jobs (to define one specific recurrence pattern value to use). See Configuring Job Schedule Frequency for more information.

  1. Type the Recur every value into the text box and select associated time units from the drop-down menu for the currently selected Job Schedule. Options are week(s), month(s)or year(s).
  1. If the recurrence interval unit is month(s), select a Required by day within the month for the job to be complete.
    • Click Specific day of month and type the number value of that day into the text box to require the jobs to be complete by a specific day of each month.
    • Click End of month to require the jobs to be complete by the end of the last calendar day of each month.
  1. Type the Available for value into the text box, and select associated time units from the drop-down menu. Options are week(s), month(s)or year(s).
  • Click Variable to access the property fields required to configure a variable, data-based frequency for the scheduled jobs (to define data threshold values that will determine which of a variety of possible recurrence patterns to use). See Configuring Job Schedule Frequency for more information.

  1. Select a Data type from the drop-down menu and also select its associated units.
  2. Define the Data averaging period. Type in the number of months to use when averaging data for determining threshold values.
  3. Configure or edit the data threshold table as required. See Adding or Deleting a Variable Data Threshold for more information.
  1. When all options are configured, click OK to save the new Job Schedule settings.

Deleting a Job Schedule

Use the Delete button to remove existing job schedules. When a job schedule is deleted from the list, all jobs associated with that schedule, if they are not yet in a 'completed' state, are deleted as well.

Ensure that the selected job schedule, and any incomplete jobs scheduled using it, are no longer needed before deleting a job schedule from the system. To delete, select a job schedule in the list, and then click Delete to remove it. A confirmation dialog appears to make sure you want to make the deletion.
