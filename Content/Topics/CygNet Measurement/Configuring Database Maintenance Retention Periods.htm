Configuring Database Maintenance Retention Periods

Use the Database Maintenance page of the Extended Command Options dialog box to configure retention period details for command and transaction logs. These retention period values must be configured prior to running the database maintenance command.

The command can also be initiated by scheduling the Perform Database Maintenance command in the Master Scheduling Service (MSS) in CygNet Explorer (most common usage), or via scripting.

Note: The definition of the command resides in FMS, but you can also use CygNet Explorer (MSS service) to schedule this command. Scheduling a Database Maintenance command in the MSS will ensure that maintenance tasks are included in system scheduling. See Scheduling FMS Command Tasks in the MSS for more information.

You must have proper security authorization to configure database maintenance retention periods. See FMS Security (and CMDSYS security event) for information about configuring security access to extended command options.

See Running Perform Database Maintenance Commands for more information about running the configured command in FMS Explorer.

See Perform Database Maintenance for more information about additional options for running the command.

Database Maintenance Command Retention Periods

The Database Maintenance commands that are set up to keep your CygNet Measurement database in good working order reference configured retention periods for UIS/FMS transaction logs and for each type of command in your system.

Configuring Database Maintenance Retention Periods

Use the following steps to configure transaction log and command retention periods for FMS Database Maintenance commands.

To Configure Database Maintenance Retention Periods

  1. Open the Commands menu in FMS Explorer, and then click Extended Options to access the Extended Command Options dialog box.
  2. Click Database Maintenance to access the database maintenance extended options configuration pane.
  3. Select and define the following extended options to configure settings for database maintenance.
    OptionDescription
    Retention period for UIS/FMS transaction logs

    Type in the period of time for which UIS/FMS transaction logs will be retained, in number of days, in the text box.

    Default value is 15 days.

    Retention period for each command type

    Type in the period of time for which command logs will be retained for each Command type, in number of days, in the Days column.

    Default value is 30 days.

  4. When all options are configured, click OK to save the settings.
