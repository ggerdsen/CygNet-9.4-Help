Migrating Files from GMR into FMS

Notes:

  • CygNet Gas Measurement Repository (GMR) has been discontinued. Effective with the CygNet v8.5.0 release, GMR files must be migrated to FMS.
  • CygNet Measurement (FMS) provides GMR-equivalent functionality (in REPOSITORY mode) or additional functionality (in FULL mode) for managing electronic flow measurement (EFM) data. Devices previously supported only by GMR data groups are now supported in FMS using FMS Legacy data groups only. Devices previously supported by both GMR and FMS data groups are now supported in FMS using FMS data groups only.
  • Refer to prior versions of the CygNet Help document for information applicable to GMR systems.

Use the GMR Migration tool in the FMS Toolbox to accomplish the tasks required to migrate files from a Gas Measurement Repository (GMR) service into a CygNet Measurement (FMS) service. The following is an overview of the component migration tasks you can perform utilizing the FMS Toolbox.

Prior to migrating your files from a GMR service to an FMS service, assess your needs and complete any preparation tasks to ensure that the migration tasks go smoothly. See Converting to CygNet Measurement from GMR for more information.

Ease and success of a large scale migration is dependent on the quality of the data files involved. Familiarize yourself with the process, the required data items for your device types, and possible errors that might occur, and take steps to correct any potential problems before you begin.

You must have proper security authorization to perform various aspects of file migrations. Required security levels are noted in the applicable steps.

See Required Device Configuration Data for CygNet Measurement for more information about device data item configuration.

See Troubleshooting CygNet Measurement for more information about potential problems.

Resetting Logging Keywords

Prior to starting the migration from GMR to FMS, reset your logging preferences to ensure that the Event Logging Service (ELS) is not overloaded by migration process exceptions.

To Reset Logging Keywords Prior to GMR/FMS Migration

  1. On your host server, find the CygNet\Services\FMS folder and open the FMS configuration file fms.cfg.
  2. In the Logging Keywords section, find the keyword LOGMASK_ELS and remove EXCEPTIONS, if it is listed as one of the values.
  3. Save the file.
  4. Stop and restart the FMS Service for the change to become effective.

Once the migration from GMR to FMS is complete, reset your logging preferences back to the level desired.

See Service Configuration Files and FMS Configuration File Keywords for more information about making configuration file changes.

GMR Migration

The following steps help you migrate files from GMR into CygNet Measurement using the GMR Migration tools from the CygNet FMS Toolbox.

  • Export Devices - Use the utility to export GMR device data and process variables from GMR to .csv files.
  • Export Groups - Use the utility to export GMR group data from GMR to .csv files.
  • Configure and edit device .csv files. - Prepare exported device .csv files for import.
  • Configure and edit group .csv files. - Prepare exported group .csv files for import.
  • Import Devices - Use the utility to import device .csv files into FMS.
  • Import Groups - Use the utility to import group definition and/or group entries .csv files into FMS.
  • Migrate Events - Use the utility to migrate events, alarms, and configuration information from GMR to FMS.
  • Migrate Contract Hour - Use the utility to migrate contract hour information from GMR to FMS.
  • Migrate History - Use the utility to migrate history data from GMR to FMS.
  • Migrate Data Groups - Use the utility to migrate data groups (for Emerson device types) from GMR to FMS.

Note: You can automate certain GMR Migration operations (Events, Contract Hour, and History) using a third-party scheduler such as the Microsoft Windows Task Scheduler. See Start the application from a command line in the Installing and Starting FMS Toolbox topic for more information.
