Running File Export Commands

Use the File Export button to access a sub-menu of available file export command types. Each command type triggers the service to search a specified directory and export data files corresponding to the dates and parameters referenced by the selected command. Command success or failure is saved in the command log.

File export commands take the form Export: X, where X represents the specific export file type available on your system. Each file type equates to a single command, e.g. Export: PPAs CSV is a command to export prior period adjustment .csv files.

See the corresponding configuration topics for more information about configuring export commands for specific types of export files. Available file export commands may include the following types.

  • PPAs CSV (only available in systems licensed for FULL mode)
  • Normalization View Data CSV (only available in systems licensed for FULL mode)

Note: To access optional file export command functionality, if installed, use the Node Commands buttons on the Commands menu. See Running Node Commands and Managing Node Commands for more information.

See FMS Commands for more information about running this command via CygNet Explorer.

Running a File Export Command

Once the file export command is configured as desired, click Run command to run the selected Export File command manually. Command results will be shown in the Command Results pane of the FMS Command dialog box. Click on a row in the Node list (top pane) to display details in the command results list (bottom pane). Execute a separate command for each file type you want to export.

Note: For best system performance, it is strongly recommended that multiple file exports performed for the same Node(s) are executed in chronological order.

File export commands can also be initiated by scheduling the command in the Master Scheduling Service (MSS) in CygNet Explorer, or via scripting.

See Scheduling FMS Command Tasks in the MSS for more information about scheduling commands.

See CxFms for more information about scripting commands.
