Importing GMR Group CSV Files into FMS

The Import Groups task helps you to import group definition and/or group entries files into FMS to create FMS group Nodes. After the group .csv files containing your GMR information have been configured and edited, the next step of the migration process is to import the prepared files into FMS, using this task. You can import group definition files and group entries files as a single action or as separate actions.

You must have proper security authorization to import files into FMS. FMS ACCESS event authorization level 5 is required to complete this step. See FMS Security (and ACCESS security event) for more information about security event authorization.

Importing GMR Groups

To Import a Group CSV File into FMS

The following procedure applies to importing general or station groups into FMS.

  1. Start CygNet FMS Toolbox, and then click GMR Migration to access associated tasks.
  2. Click Import Groups to access the Import Groups pane.

  3. Enter information into the following fields. Definition file name or Entries file name may be left blank if you only want to import the other file type with this action.
  1. Definition file text box. To import group definition files, type the name of the group definition file you wish to import into FMS, or click ... to access the Select Exported Group Definition File dialog box to browse to your selection and then click Open.

    Note: You may select previously imported .csv files for reimport, in cases where there were previously errors in some of the groups and you want to reimport those groups after editing the .csv manually to provide proper parameters. The reimport will skip groups that were previously imported successfully (for which "FmsImportStatus" = OK).

  2. Entries file text box. To import group entries files, type the name of the group entries file you wish to import into FMS, or click ... to access the Select Exported Group Entries File dialog box to browse to your selection and then click Open.
  3. FMS text box.
    • To import files to a Site.Service located on the same domain, type the name of the FMS Site.Service to which you wish to connect to import the file, or click ... to access the Select Import FMS dialog box to browse to your selection and then click OK.
    • To import files to a Site.Service located on a different domain, type [XXXX], where XXXX is the name of the domain, followed by the name of the FMS Site.Service to which you wish to connect to import the file.
  4. Audit category drop-down menu. Select an audit category from the options in the drop-down menu. This menu will be populated based on the FMS Site.Service selected. Audit categories are configured on the Admin menu. See Auditing in CygNet Measurement for more information about creating audit categories.
  5. Audit comments text box. You have the opportunity to type Comments regarding the file migration into the text box, which will become part of the audit record for the file import. Although comments are optional, they can be helpful when auditing or reviewing records later.
  1. Click Import to import the selected .csv files into FMS. Import status, including any import errors, will be displayed in the text box above the Import/Restart button.
  2. If any import errors have occurred, do the following.
  1. Go back and edit the group .csv files manually to correct any import errors.
  2. Once errors have been corrected with proper parameters, click Restart to reimport the files.

    Note: For group .csv file imports, the Restart process will skip groups that were previously imported successfully (for which "FmsImportStatus" = OK) and will import the corrected groups.

  3. Repeat steps a - b until the import is successful.
