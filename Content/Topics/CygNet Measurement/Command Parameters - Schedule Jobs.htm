System: Schedule Jobs

Note: This command operates in conjunction with CygNet Dispatch. For more information about the CygNet Dispatch product, contact your Account Manager or CygNet Sales.

Schedule Jobs (SCHEDJOB via script) is a system-level command that triggers the service to automatically produce scheduled jobs for use in CygNet Dispatch, based on configured parameters, data value ranges, and industry requirements, for the specified group or device Nodes. Job scheduling results in the generation of recurring jobs that are added to the scheduled jobs list in CygNet Dispatch upon data synchronization.

Note: This command is available only via the Master Scheduling Service in CygNet Explorer or via scripting (not through the FMS Explorer user interface).

You must have proper security authorization to execute this command. See FMS Security Events (and JOB security event) for information about configuring security access for FMS commands.

See Scheduling FMS Command Tasks in the MSS for more information about scheduling commands.

See CxFms for more information about scripting commands.

Managing Job Scheduling Commands

Prior to running Schedule Jobs commands, define the command parameters, and configure job schedule information in the extended command options, via the FMS Explorer Commands menu. See Configuring Job Schedules for more information.

When the Schedule Jobs command is run, jobs are created according to the job schedule associated with the Node, and those jobs are made available to CygNet Dispatch when it is synchronized with the FMS service. See CygNet Dispatch for more information.

Command Definition Properties

The following command definition properties apply to Schedule Jobs commands, when scheduling, creating, or editing commands via the Master Scheduling Service (MSS) in CygNet Explorer.

Property Description
FMS service Select a Site.Service for the command from the available FMS service list in the drop-down menu.
Command

Select the desired Command name from the drop-down menu.

See FMS Commands for more information about available command types.
Priority

Select a processing queue Priority for the command from the drop-down menu.

Options from least to greatest priority are as follows.

  • Low level
  • Medium level
  • High level
  • User initiated transaction
  • Admin initiated transaction

Default value is Low level processing queue priority.
Logging

Select a Logging level for the command results from the drop-down menu.

Options are as follows.

  • Low - displays errors
  • Medium - displays errors and warnings
  • High - displays errors, warnings, and status

Default value is Low level logging when scheduling a command via the MSS service in CygNet Explorer.

Default value is High level logging when creating or running an ad-hoc command in FMS Explorer or via the FMS service in CygNet Explorer.

Command Parameters

Note: Additional Schedule Jobs options are defined through the FMS Explorer Commands menu via the Extended Options button, prior to running the command. See Configuring Job Schedules for more information.

The following command parameters apply to Schedule Jobs commands, when scheduling, creating, or editing commands via CygNet Explorer or scripting.

Parameter Description Type Required? Usage Notes
SJName Job schedule name STRING Optional

If used, enter the name of the job schedule configuration to use when executing this command.

Double-click to access the Enter Job Schedule Name dialog box, and type a name into the text box.

This parameter is optional. If not provided, the default value is * (multiple character wildcard); run the command for all job schedule configurations.

When a group Node is selected, use the Device descriptor(s) section of the dialog box to further filter the group to select the member Nodes to include in the Schedule Jobs action.
