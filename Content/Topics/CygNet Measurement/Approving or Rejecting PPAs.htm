Approving or Rejecting PPAs

Use the Prior Period Adjustment (PPA) control to approve or reject PPAs from the control data grid.

Once created, PPA records must be approved or rejected for export, and a record of the action will be retained for the audit trail. See Configuring Systems Options for more information about administrative tasks required to configure the manner in which PPAs are handled.

You must have proper security authorization to handle PPAs. See FMS Security (and PPA security event) for information about configuring security access for working with PPAs.

Approving or Rejecting a Prior Period Adjustment

Right-click on a record row in the PPA control grid to access the context menu.

The context menu contains the following options to help you manage PPAs.

Option Description
Approve for export

Click Approve for export to access the Save record dialog box.

You will need to select an Audit category from the drop-down menu, and you will have the opportunity to write Comments regarding the record in the text box. Click to Save the Prior Period Adjustment record approval.

See Configuring Audit Categories on the Admin menu to add audit categories. For Category type select Handling PPAs.
Reject for export

Click Reject for export to access the Save record dialog box.

You will need to select an Audit category from the drop-down menu, and you will have the opportunity to write Comments regarding the record in the text box. Click to Save the Prior Period Adjustment record rejection.

See Configuring Audit Categories on the Admin menu to add audit categories. For Category type select Handling PPAs.
Set as Session Node

Click Set as Session Node to set the selected Node as the Session Node, to consult additional data.

Alternatively, press Ctrl+Shift+S to set the Session Node using a keyboard shortcut.

See Session Node and Using Keyboard Shortcuts for more information.

Decimal precision

Click to select Decimal precision for the display from 0 - 10 decimal places.

Default value is 7

To Approve or Reject PPAs

  1. On the PPA control Options pane, make the following selections.
  1. Select the time window and time format for which you want to view PPAs. Either select to Use Session options, or select the desired start date and end date, and the time format for the PPAs you want to view. See time format in the control Options table for more information about time selection details.
  2. Select a Unit set from the drop-down menu.
  1. If you want more information about the data, click to select a single PPA record from the data grid for which to view additional data. Right-click on the selected Node to access the context menu, then click Set as Session Node to investigate it further using additional controls. See CygNet Measurement Workflow for more information.
  2. On the PPA control grid, click to select one or several PPAs to approve or reject for export.
  3. Right-click to access the context menu options and click to select one of the following
  1. Click Approve for export to access the Save Record dialog box.
  2. Click Reject for export to access the Save Record dialog box.
  1. On the Save Record dialog box, do the following.
  1. Select an Audit category from the drop-down menu.

    Note: See Configuring Audit Categories on the Admin menu to add audit categories. For Category type, select Handling PPAs.

  2. Enter Comments in the text box regarding the approval or rejection of the PPA. Although this step is optional, it can help identify the record later.
  3. Click to Save the PPA record.
