Configuring Audit Categories

In many places within the system, an internal audit record is attached to create a detailed audit trail when you make a change. Use the Audit Categories page to manage a list of available audit category options, per audit type, in order to help you track and understand changes occurring in your system. You can use the default categories provided, or edit or add category names to create customized categories. When you save the changes, the categories will appear in the Audit category selectors throughout the system.

You can also access audit history information from this page. See Viewing Audit History for more information.

You must have proper security authorization to configure audit categories. See FMS Security (and ACCESS security event) for information about configuring security access to administrative options.

See Auditing in CygNet Measurement for more information about internal auditing in the measurement system.

Viewing or Creating Audit Categories

On the Admin menu, click Options, then Audit Categories to access the audit categories configured for your system.

The properties for the Audit Categories page of the Administrative Options dialog box are described below.

Parameter

Description
Category type

Select the desired category type from the drop-down menu to view its audit categories. Default audit categories are provided for each category type.

You can also create customized categories within any category type. Either select and edit an existing audit category name, or click Add to insert a new category listing and then type in the new audit category name desired.

Options are as follows.

  • Configuration Change — This audit category type is used for configuring Node properties on the Node Chooser, Dashboard, and Exceptions controls. It is also used for changes to Validation or Estimation engines.
  • Data Change — This audit category type is used for changing data on the History Grid control.
  • Handling PPAs — This audit category type is used for approving or rejecting PPAs for export on the Prior Period Adjustment control.
  • Ignoring Exceptions — This audit category type is used for ignoring exceptions on the Exceptions control.
  • System Event — This audit category type is used for closing a period on the Close Period control.
