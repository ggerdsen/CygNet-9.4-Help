Supported Tokens for Device Data CSV Exports

In general, tokens supported for the export definition file used to define Device Data CSV exports are equivalent to the data item tokens (Data Table Tokens) supported for Device QTR, Gas Device Raw QTR, or Liquid Device Raw QTR reports, but there are differences in both content and format. Some tokens are supported in one format and not the other. Also, whereas the QTR reports recognize valid token names enclosed in brackets [TokenName], the EDF files recognize equivalent valid token names enclosed in quotation marks "TokenName".

Additionally, different token sets are supported depending on whether you are exporting normalized or raw (gas or liquid) data. For more information about valid token names for normalized or raw data, see the following:

  • For normalized data, see the Normalized Data Tokens table below, or see tokens listed for:
  • Configuring the Report Sheet in the (Gas) Device QTR Report Templates topic
  • For raw data, see the Raw Data Tokens table below, or see tokens listed for:
  • Configuring the Report Sheet in the Gas Device Raw QTR Report Templates topic
  • Configuring the Report Sheet in the Liquid Device Raw QTR Report Templates topic

Example

Valid token for QTR reports = [TokenName]

Valid token for EDF = "TokenName"

See Export: Device Data CSV for more information about preparing and exporting .csv files that utilize an .edf file.

Displaying Beginning or Ending Values for a Record

Some data item tokens (for normalized data) have variants you can use to display the beginning or ending values for a record. Append _Begin to the token text to show the beginning value for the record period. Append _End to the token text to show the ending value for the record period. You can then view data values or configuration settings in effect at either the beginning or end of each record span (e.g. hourly, daily) as desired. Examples of valid appended tokens are listed in the Configuration Item, Gas Quality Item, and Liquid Periodic Metering PV Item sections of the tables below.

Displaying Flow-Weighted Average Values for a Record

Some tokens for gas metering process variable values (for normalized data) have a variant you can use to display the flow-weighted average value for records in your report. Tokens utilizing this version are valid for [Energy_Average_FW], [Mass_Average_FW], [Volume_Average_FW], and are listed in the table, below.
