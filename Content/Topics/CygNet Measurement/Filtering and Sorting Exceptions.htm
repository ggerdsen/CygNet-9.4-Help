Filtering and Sorting Exceptions

Use the Exceptions control to quickly filter and group exceptions and the manner in which they are displayed, using the column filtering row in the data grid (below the column header row), and an optionally displayed custom grouping hierarchy (above the column header row).

Note: You can also optionally configure and filter what sorts of exceptions to show in the Exceptions control, using the filter available on the control toolbar. See Toolbar for more information.

Exceptions can be filtered and grouped into any hierarchy desired, and the hierarchy can quickly be changed again for the next task. Once the exceptions data is organized, you can "drill down" to check the exceptions at various levels within the hierarchy created, using the expand (+) and contract (-) icons. When an exception requiring further attention is identified, it can be set as the Session Node, to help further analyze its associated data in order to make informed decisions about handling the exceptions.

Once an exception is analyzed, it can be ignored if appropriate. See Ignoring Exceptions for more information.

Filtering and Sorting Exceptions by Column Values

Below each column header in the data grid is a text box that can be used to filter exceptions data by a value you enter.

To set a filter, click to select the text box below the desired column, then type the filter value into the text box. The default filter is <all>. You may enter values in multiple columns. An arrow appears within the column header, indicating the column that is currently selected for filtering, and the order in which the results are shown. Click the arrow to toggle the order selected.

To clear the filter, click the close button (x) in the text box.

Sorting Exceptions by Custom Grouping Hierarchy

Above the column header row in the data grid is an optionally displayed group box. You can use this feature to further sort exceptions data hierarchically, by groups you select, in order to clarify the data display.

Right-click on any Exceptions grid column header to access the context menu options to display/hide the Field Chooser and/or Group Box. These tools can both be used to configure a custom grouping hierarchy with which to efficiently sort and view exceptions data.

Select Field Chooser to display a list box in which to "store" and access the property column headers that are not currently displayed in the grid. Although these property columns are not part of the data currently being viewed, they can still be part of the data sorting process. See Using the Field Chooser for more information.

Select Group Box to display a box above the column header row in which to organize the group filtering hierarchy you configure. To create the grouping hierarchy, drag desired column headers to the group box, either from the column header row or from the field chooser. To change the grouping hierarchy, drag any component out of the group box or to a new position within it. Red positional arrows appear to guide placement of the new grouping element.
