Export Tools

Export Tools

CygNet Software supports many options for exporting data and system configuration settings from its services, applications, tools, and controls. The following list directs you to many of these tools.

CygNet Feature or Application Usage
CygNet DDS Import/Export utility Export DDS Facility Header Records to a .csv file.
CygNet DDS Viewer utility Export transaction headers or data for a selected time period for a selected communication device, import/export device, or data group.
CygNet Dynagraph Viewer/Dynagraph Control

Export a card from the CygNet Dynagraph Viewer or from a Dynagraph Control on a CygNet Studio screen.

Export a set of dynagraph cards on a routine basis using the CxDynagraph.CxExport object.

CygNet Excel Export

CygNet Excel Export (CxExport.dll or CxExport64.dll) can be used to create reports in a variety of applications, including Microsoft Excel, VBA, C++, Word. A sample use of the .dll with Excel is included, which can be used to generate the following reports directly with Microsoft Excel.

  • History Values Report
  • Current Values Report
  • Current Values by Group Report
CygNet Explorer, Service Details

Export the contents of the current Service Details grid to a .csv file.
CygNet GCM Nomination Grid

Export General Contract Monitor nomination grid values to a .csv file.

CygNet Measurement, FMS Commands

Data can be exported from CygNet Measurement using the following FMS commands.

  • Export Normalization View Data
  • Export Prior Period Adjustments
  • Export: Flow-Cal Data CFX (if installed)
  • Export: Flow-Cal Ticket Data TFX (if installed)
  • Export: Flow-Cal Transaction Queue (if installed)
  • Export: PGAS Data XML (if installed)
CygNet Measurement, FMS Controls

Data can be exported from CygNet Measurement controls in the following ways.

  • Export measurement data from FMS grid controls via CSV files
  • Export History Graphs in a variety of graphical export formats from the FMS History Graph control
CygNet Measurement, FMS Reports

Run and email or export a variety of industry-standard FMS Reports in a choice of file formats
CygNet Reports

Export CygNet data using CygNet Reports. Reports include Alarm Activity, Communications Efficiency, GNS LOG, Morning Report, and Daily Summary.
CygNet Service Diagnostics utility

Export Info Items data to a .csv file.
CygNet Studio Controls

Data can be exported from the following CygNet Studio objects.

  • Event Summary Control - Export the contents of the Event Summary grid to a spreadsheet.
  • Group Grid Control - Export the contents of the Group Grid control to a spreadsheet.
  • Nested View Tool - Export the data in the dictionary to an XML string.
  • Trend Tool - Exports the contents of the trend to a spreadsheet.
  • Watchlist Tool - Exports the data in the dictionary to an XML string.
CygNet Version Info utility

Export file names and current version information to a text file.
Devices

Export device configuration from the DDS using CygNet Explorer.
EIEs - Import/Export Devices

The following Import/Export EIEs are available for exporting data from CygNet Software.

  • Enterprise Operations EIE (EntOps)
  • General Contract Monitor EIE (GCM)
  • Line Pack EIE (LPack)
  • Sonic Queue EIE (SonicQ)
  • Text Import EIE (TextImport)
Energy Load Forecasting

Export ELF device settings to an .xml file.
Enterprise Operations

Enterprise Objects provides an interface between an Enterprise Data model and the CygNet Native (SCADA) Data model. This interface facilitates the exchange of data between the CygNet Software platform, third-party applications, and other business systems. Additionally you can do the following.

  • Export a Business Object Package file from the CygNet Business Object Builder.
  • Export EntOps device settings to an .xml file.
Facility Load utility

Export Facility records to a .csv file.
Filter Rules

Export configured filter rules to an .xml file.
Group Service Explorer

Export GRP attributes for a selected node to an XML file.
History Values

Historical data can be exported from the VHS with several different tools.

  • Export historical data with CygNet Excel Export.
  • Export historical values for a point from the History Values dialog box.
  • Export historical data from the VHS to a .csv file with the CygNet History Export utility.
  • Export data analysis displayed in the view to a .csv file with the VHS Data Analyzer utility.
  • Export data from the VHS with the VHS Import/Export utility.

License Manager utility

Export license manager data to a .csv file.
ODBC

Data can be retrieved from CygNet using the CygNet ODBC driver for use with an ODBC application such as Microsoft Access, Microsoft Excel, or Crystal Reports.

CygNet Reports uses ODBC to retrieve data.
Point Configuration Manager utility

Export points to a .csv file.
Service Diagnostics utility Export configuration settings to an .xml file.
Scripting Export Options

Data can be exported using the following automation interface methods.

  • CxDds DdsClient.ExportDevices
  • CxExport.ExportClient
  • CxHmiGrp CxGrpGrid.Export
  • CxScript Dictionary.ExportToXml
  • CxVhs.VhsClient
Trend Tool

Export trended data from a Trend object on a CygNet Studio screen or other location within CygNet.
TRS Load utility

Export TRS tables to a .csv file.
