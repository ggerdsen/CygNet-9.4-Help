CygNet Excel Export Error and Status Messages

Excel has a limit of 256 columns and 65,536 rows. If your report exceeds these thresholds, you will encounter errors.

If Display Status Messages is enabled in the Export page in the Setup dialog box, a message will appear indicating the success or failure of the report.

Message Description/Resolution

Read tag file c:\directory\file name.txt or file name.csv

Uses an imported tag list in the form of a text file or .csv.

Export to file: c:\cygnet\support\exportName

Exported the file to this directory.

Exported xx (of xx) Values for SITE.SERVICE.tag_string_format

Shows the number of exported values for that Site.Service and point.

Export Complete

The export is complete with or without errors.

Send messages to services to update local cache

This activity validates that the values from the CygNet services residing in temporary memory (or "cached values") are up-to-date for the export. To see export activity messages, you must enable Display Status Messages on the Setup dialog box Export page.

Error in export thread

There is an error or errors in the export. If there is no output, then there is a problem with the CygNet Excel Export configuration. Check the user-defined report parameters in the Setup dialog box and generate the report again.

Unable to Resolve MYSITE.SERVICE::*.UDC

A valid point was not exported because the point might not be associated with a specified site or facility named in the Setup dialog box.

No valid tags to export

A valid tag, or point, was not exported because it did not exist for a facility or a group node. When configuring the Current Values by Group Report, check the hierarchy and make sure that the configured tags are correctly associated with the right sub-node. Try running the report  again and leaving the Sub-Node blank.

Error Exporting … returned error (xx) 'Not a history point'

A point that might be valid for the facility is not a history point, for it does not reside in the VHS. Check the point or tag in the point configuration record.

“Facility attributes, Site, Service and Timestamp columns must precede any other”

Site, Service, Timestamp, and all Facility-based attributes must precede any other (point-specific) attributes when Facility is selected in the Format page. Check the Export Columns in the Columns page and make sure that facility items are at the top of the column list.

Tag Specification method is invalid for this data source

Tags, or points, are not being generated. Check the Tags page in the dialog box and make sure that the tags, or points, are correctly listed.
