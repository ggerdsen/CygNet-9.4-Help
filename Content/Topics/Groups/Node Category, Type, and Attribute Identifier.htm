Node Category, Type, and Attribute Identifiers

The node category, node type, and node attribute identifiers define a given node. They appear in node properties dialog boxes, the GRP service pane, and the Group Service Explorer.

Note: Multiple GRP nodes might result from adding a single node to the Group Manager utility. In such a case, the multiple GRP nodes represent multiple roles of the Group Manager node.

Node Categories

The following table lists the abbreviated IDs, long names, and brief descriptions for all available node categories.

ID Long Name Description

R

Root Group

The root group represents the root node of one or more actual hierarchies.

Note: The component hierarchy root is not a root group node; it is an administrative node (Z). Actual hierarchies are defined beneath the Hierarchies administrative node. Navigation hierarchies define actual hierarchies.

S

Sub Group

 A subgroup node is subordinate to a root group node, but it is not a leaf node.

L

Leaf Node

A leaf node is the end node in a hierarchy branch; there are no nodes beyond the leaf node. A leaf node usually represents the source of data entering a system or hierarchy.

In the context of CygNet groups, the leaf node is the only node in a hierarchy that is explicitly associated with a facility. Other nodes may reference a facility (for use in group rollups, for example), but only leaf nodes require a facility.

Z

Admin Node

An administrative node represents a system-generated, organizational node within a hierarchy, but is not itself a data-producing node.

Examples include a component hierarchy root node, which represents an organizational root for all hierarchies and components in a group, and a Summary node, which is generated by the system whenever you add a view.

Node Types

The following table lists the abbreviated IDs, long names, and brief descriptions for all available node types.

ID Long Name Description

~G

Generic

 Catch-all type for nodes that do not match the criteria of other node types.

~U

Summary

 Defines a node as the point at which a summary of values for all associated attributes within a single view is to occur.

~F

Facility

Defines a node as one that relies on a facility for its property values and attributes.

~D

Device

Defines a node as one that relies on a device for its property values and attributes.

~P

Point

  

~W

Web Root

Defines a node as a root for an individual Web navigation hierarchy.

~V

Navigation Root

Defines a node as a root for an individual navigation hierarchy. Multiple entries occur if multiple views use the same navigation root.

~L

Component Def Head

 Component definition header. Encapsulates all component nodes within a specific component hierarchy.

~B

Component Def

 Component definition. Defines an individual component node. Appears under the Components node in the Group Manager.
~T Fac Type Def Head Facility type definition header. Defines a node as the parent of a Facility Type List used for a Web view.
~C Fac Type Def Facility type definition. Defines a node as the child of a Fac Type Def Head node. Represents one Facility Type used by a Facility Type List for a Web view.

~J

Attribute Def Head

 Attribute definition header.

~A

Attribute Def

 Attribute definition.

~K

Hierarchy Def Head

Hierarchy definition header. Encapsulates all administrative (node category Z) nodes within a specific view.

Note: When you create a view, the Group Manager utility and Group Service (GRP) create two nodes to represent different roles of one view. One GRP node defines the view as category (Z), type (~K) and the other as category (R), type (~G).

~H

Hierarchy Def

 Hierarchy definition. Defines a node of a (hierarchy) view, represented as a level in the Group Manager utility.
~Q Report Def Head Report definition header.
~R Report Def Report definition.
~S Storage  

Node Attribute Identifiers

The following table lists the abbreviated IDs, long names, and brief descriptions for all available node attribute identifiers.

Note: The node attribute identifiers described below appear in various places in the node property sheets accessed from both the Group Service (GRP) and Group Manager utility. They are unique to the GRP; do not confuse these with facility and device attributes.

ID Long Name Description

~cat

Category

 The node category for a selected node.

~type

Type

 The node type for a selected node.

~nodeid

Node ID

 System-generated number that uniquely identifies a single node.

~desc

Description

 Free-form description of a node.

~ref_type

Reference Type

 The reference Type for a reference entity associated with a node. Select the appropriate option from the Type drop-down menu.

~ref_site

Reference Site

 The Reference Site for a reference entity associated with a node.

~ref_serv

Reference Service

 The reference Service for a reference entity associated with a node. Typically a UIS.

~ref_id

Reference ID

 The ID of the item referenced by a reference entity. Available selections depend upon the Type chosen.

~security_app

Security Application

 Name of the security application that applies to a node. Appears in the ACS Application field of the node property sheet.

~security_event

Security Event

 Name of the security event that applies to a node. Appears in the ACS Event field of the node property sheet.

~hier_root

Hierarchy Root

 Identifies the node ID/data key of the selected node's hierarchy root node, also known as type Hierarchy Def Head (~K).

~attr0 - 21

Attribute 0 - 21

 Displays various node-specific attributes from 0 to 21. These are attributes with alphanumeric string values.
~attryn0 - 19 Yes/No Attribute 0 - 19 Displays various node-specific yes/no attributes from 0 to 19. These are attributes with only a "Yes" or "No" value.

~appl0 -

Application 0 - 5

 Displays various node-specific applications from 0 to 5.

~visible

Node Visible

 If the Visible check box is checked in the property sheet for a node and/or if the ~visible value is "1", the node's record appears in the GRP.

~auto_maintain

Auto Maintain

 Obsolete
~aux_data Has Aux Data Displays a Boolean value that indicates whether or not a node has auxiliary data.
~aux_length Aux Data Length The length in bytes of a node's auxiliary data.
~aux_crc Aux Data CRC Displays the CRC of a node's auxiliary data.
~aux_timestamp Aux Data Timestamp Displays a timestamp of the last time auxiliary data was edited and saved.
