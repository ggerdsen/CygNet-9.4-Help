Error Messages

The following error messages indicate the kind of error that occurred in either a group rollup HyperPoint (HSS) or calculation target point (UIS). Checking the error status of a newly created rollup HyperPoint or calculation target point helps to ensure that you have correctly configured the point(s).

Note: If an error occurs, no value is posted to the calculation target point unless the error is in one of the following states: UNRELIABLE or NON-NUMERIC.

You have the following options for viewing group rollup errors:

  • Using CygNet Explorer, open the HyperPoint Scripting Service (HSS), locate the HSS point you are interested in, and look at the value in the Value column.
  • Using CygNet Explorer, open the Point Service (PNT), locate the point you are interested in, right-click the point, click Current Value, and look at the value in the Value field.
  • Using CygNet Explorer, open the Universal Interface Service (UIS), click the Current Values page, locate the UDC you are interested in, double-click the UDC, and look at the value in the Value field of the Point Properties dialog box.
Error Description
No UDC No UDC for the rollup was specified.
No Facility No facility for the rollup was specified.
Failed Connect The GRP failed to connect with the UIS to retrieve data.
Bad Rollup Type

The selection of the Point Reference Type field is not applicable.
FacInfo UDC Err The alarm rollup requires the use of SYFACINFOR as the UDC.
No node match

There is no Summary (category "~U") group node in the hierarchy with the correct facility name.

>1 node match Multiple summary nodes have the same facility name.
[Node ID #] Bad The node ID is incorrect. [Node ID #] represents the bad group node ID.
Div by Zero No points found for a Group Averager rollup type.
Setpoint Failed The target tag does not exist or security does not allow the setpoint task.
UNRELIABLE One or more points are unreliable. When this occurs, the calculation is still posted to the calculation target point, but the target point is set to UNRELIABLE.
NON-NUMERIC One or more points had a non-numeric value. When this occurs, the system assumes the non-numeric value is 0 and continues with the rollup, but it marks the data as unreliable.
Pt Not Found One of the facilities included in the rollup does not have a point for the rolled-up UDC.
