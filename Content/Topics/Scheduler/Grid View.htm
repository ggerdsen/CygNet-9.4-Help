Grid View

The Grid view of the MSS Viewer utility enables you to create a customized, hierarchical view of configured tasks. Numerous task fields are available to choose from and may be organized in a way that suits your viewing needs. This functionality enables grouping and sorting of existing tasks to better troubleshoot scheduling issues. For instance, a hierarchical view of tasks makes it much easier to locate resolved-to devices.

Configuration changes may be made on an ad hoc basis on a per-task basis; these changes are then pushed to the Master Scheduling Service (MSS). Once you've configured a grid to display tasks in the format you want, the grid format persists after closing and opening it, after an MSS shutdown, and after an upgrade.

Configuring a Grid View

The following procedure provides a basic workflow for setting up an MSS Viewer Grid view. It is designed to give you an overview of possibilities for your grid. Some unique user interface features are described that help to provide a very highly configurable view.

To Configure a Grid View

  1. After opening the MSS Viewer utility, click View and select the Grid option.
  2. Select the appropriate MSS using one of the following methods:
    • In the toolbar, click and select the appropriate MSS.
    • In the toolbar, click the Browse button to the right of the Site.Service field and select the appropriate MSS.
  3. After all of your unsorted tasks appear in the default view, create hierarchical groups.
    1. Enable the Group Box by right-clicking in the default column header and clicking Group Box. A large blank gray field appears at the top of your grid.
    2. To select fields other than default fields by which to sort, right-click the Group Box and select Field Chooser.
    3. Add fields by dragging and dropping them from the Field Chooser onto the column header in the position you require.
    4. Delete fields by dragging and dropping them from the column header into any space other than the column header or Group Box.
    5. When you're done adding and deleting fields, drag and drop column headers from the grid into the gray Grid Box. Doing so creates an ad hoc hierarchy that displays in the grid.
    6. Move Group Box elements to reformat your hierarchy.
    7. If you want to reset your grid to the default, right-click the column header or Group Box and select Set to Default.
  4. Once your hierarchy is established, you can sort within a group by typing the name of an item in the field directly beneath a column header. By default, <all> appears in these fields. Click the Browse button beside the field to return the grid to a pre-filtered state.

Sort based on Union or Intersection:

  • Union: Filters based on the sum of all resolved devices currently selected. For instance, if you select two tasks and the two tasks resolve to twenty devices in total, twenty devices appear here.
  • Intersection: Filters based on only those resolved devices shared in common between the tasks currently selected. For instance, if you select two tasks and the two tasks resolve to four of the same devices out of twenty total, only four devices appear here.

Context Menu Options

The following table lists and describes options available from the Field Chooser context menu.

Property Description

Field Chooser

Use the Field Chooser to select from an extensive list of possible fields by which to view and group your data.

Select field options one at a time and drag each to either the column header or the Group Box. You can rearrange field positions by dragging and dropping them where applicable.

Group Box

The Group Box, if opened, appears as a large blank gray field at the top of your grid. It remains unpopulated until you drag and drop field headings into it. Once there, you can rearrange field headings on an ad hoc basis to reformat a hierarchy.

Example

Set to Default

Select to return all field settings to the default.

Show All

Select to automatically place all possible fields in the grid header. You can drag and drop field headings to remove them from the header. Once removed from the grid, they become an option again in the Field Chooser.
