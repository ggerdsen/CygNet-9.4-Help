CygNet Facility Import/Export Utility

The CygNet Facility Import/Export (FacLoad.exe) utility provides a method to import and export Facility Service (FAC) records containing facility attribute values. Because the source file is a .csv, it can be reviewed and edited using any text editor, prior to running an import or export action. The Facility Service (FAC) must be running to use this utility.

The Facility Import/Export utility adds or updates records, but it does not delete them. It also does not affect attribute configuration information (such as attribute enabled status, or attribute name).

Note: Attributes are enabled/disabled using CygNet Explorer. Attribute names are stored on the host server in the FAC folder > MetaData.dat file.

The CygNet Facility Import/Export utility is stored in the CygNet\Utilities directory (FacLoad.exe) on the host server.

Starting CygNet Facility Import/Export

Installing and Launching

To install and launch (start) the application or utility using version management:

  • Use CygNet Client Installer to install and launch the application. Once installed you can use CygNet Client Installer to start the application.

-OR-

  • If previously installed, click Start > All Programs > CygNet > <name of the application or utility> or double-click the desktop shortcut.

Alternatively, browse to the CygNet\Utilities directory using Windows Explorer and double-click the program icon. You can also start an application or utility from a command line. These methods do not provide version management.

Syntax

The command-line syntax is as follows.

Usage: [-E(export)] or [-I(import) IMPORTPATH] FAC_SITE.FAC_SERVICE
