CygNet License Manager Utility

The CygNet License Manager utility provides a method to control the license records in your CygNet system as well as to activate your host(s). License records include Client Access Licenses (CALs), Web Control Server licenses, Web Control users, Web Status users and Web View Server licenses. The CygNet License Manager utility checks the computer-specific fields (OS Version, Computer Name, and Local UUID) between uses of CygNet to detect violation of the Client Access License (CAL).

The CygNet License Manager utility is stored in the CygNet\Utilities directory (LicenseManager.exe) on the host server.

Installing and Launching

To install and launch (start) the application or utility using version management:

  • Use CygNet Client Installer to install and launch the application. Once installed you can use CygNet Client Installer to start the application.

-OR-

  • If previously installed, click Start > All Programs > CygNet > <name of the application or utility> or double-click the desktop shortcut.

Alternatively, browse to the CygNet\Utilities directory using Windows Explorer and double-click the program icon. You can also start an application or utility from a command line. These methods do not provide version management.

The CygNet License Manager dialog box appears.

To view and edit license records, you must have Level 5 (Admin) authorization for the ARS event named CHANGE. Without this level of security, License Manager can be opened, but no changes can be made to license records.

To change host activation status, you must have Level 5 (Admin) authorization for the ARS event named CHANGE.

The buttons along the left side of the License Manager window change the display to show the different types of licensing information.
