Real-Time Services

A Real-Time service is a generic name for a service that processes data.

The Real-Time services in CygNet Software are listed in the following table.

Service Description Function

CAS

Common Alarm Service

The CAS provides a centralized repository for site alarms and alarm processing for CVSs.

ELS

Event Logging Service

The ELS database maintains a log of system events (service, operational, and external).

Note: The ELS is also a DBS-based service.

ELSALM

Event Logging Service

The LELSALM is an ELS that stores alarm and notification history.

Note: The ELSALM is also a DBS-based service.

GNS

General Notification Service

The GNS database stores and processes notifications (also known as callouts).

Note: The GNS is also a DBS-based service.

NOTE

Note Service

The NOTE database stores notes data and is associated with one or more CVSs.

Note: The NOTE is also a DBS-based service.

VHS

Value History Service

The VHS is a the CygNet Historian database.

The Value History Service (VHS) uses the same underlying data storage technology (ESE) as the other DBS-based services, but it is not considered a DBS-based service due to its unique archiving functionality.
