Export Properties

The Export property page specifies the report export options. Use the Export properties to define export file format, file type, file name, and additional parameters so that you can schedule the report to run automatically.

Configuring Export Properties

Export Format

There are five Export Format options available for the report export file format. These options are as follows.

  • Acrobat PDF (default),
  • Rich Text,
  • Microsoft Excel 5.0,
  • Microsoft Excel 7.0,
  • Microsoft Excel 8.0.

File Type

There are also numerous Export As file type options available. These options include PDF, CSV, Crystal Reports, Microsoft Excel, text, Rich Text format, Microsoft Word, HTML, and XML.

File Name

Use the Export File Name field to specify the file name and path to send the exported report. The system will add the file extension to the file name based on the export file type selected. If the path does not exist, the system will create it. If there is no path specified, the export file is saved to the currently active folder. Use the Append To File Name option to append the Date or Date/Time to the file name you have specified.

See Exporting Reports for more information about exporting CygNet Reports.

Saving Report Properties

You can save and run the report from this dialog box. Once all properties for a report are defined (Report Parameters Properties and Export Properties), click Apply and then click Save/Run to both save the report properties and run the report.

You can also save the report properties without running the report. Once all properties for the report are defined (Report Parameters Properties and Export Properties), click Apply and then click Cancel to save the report properties and cancel the running of the report.

You can save the parameters to a local drive or to a BLOB Storage Service (BSS). You must have proper CygNet security authorization to write files to a BSS.
