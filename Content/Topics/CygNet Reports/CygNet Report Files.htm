CygNet Report Files

Report templates are saved as .rpt files. The report template is the "shell" of the report. It contains the report format and shows the list of possible parameters.

Report parameters specify what data to include in the report, and are user-defined. Report parameters are saved .rpx files. Any number of parameter files can be saved for a report template.

Editing or Creating Report Template or Parameter Files

Crystal Reports is required to edit or create report templates.

To Open a Report Template File (.rpt)

  1. Open CygNet Reports (Start > Programs > CygNet > CygNet Reports).
  2. On the File menu, click Open.
  3. Select BLOB File System and click OK.
  4. In the BLOB Service list, select your Applications (APPS) BLOB.
  5. In the BLOB Folder list, select Samples.
  6. In the List Files of Type, select Crystal Report Files (*.rpt).
  7. Select the applicable report template from the list, and click OK.

When the template opens, the Properties box for the template is displayed. Parameters must be defined before the report can be run.

To Open a Report Parameters File (.rpx)

  1. Open CygNet Reports (Start > Programs > CygNet > CygNet Reports).
  2. On the File menu, click Open.
  3. Select the file system (BLOB or Standard).
  4. Select the storage location.
  5. Select the file. Report parameter files have an .rpx extension.
  6. Click Open.

Once the parameter file is open, you can run the report or edit the parameters.
