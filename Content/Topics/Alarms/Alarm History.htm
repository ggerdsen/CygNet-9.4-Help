Alarm History

Alarm history is stored in the Alarm Event Logging Service (ELSALM).

To view the alarm history of a particular point do one of the following:

  • Right-click on the alarm in the alarm view and select Alarm History
  • Right-click on a CygNet-aware tool on a CygNet Studio/Vision screen and click History, then Alarm History
  • Click Alarm History on the Point Properties dialog box

To view the alarm history of all points:

  • Use CygNet Explorer to view the Alarm Event Logging Service (ELSALM) service pane.

The Alarm History dialog box, shown when viewing the alarm history of an individual alarm, provides options for showing only particular types of alarm events.

Option Description
Events

Set /Reset

Filters the display for alarm Set/Reset events.

Clear/Force Clear

Filters the display for alarm Clear/Force Clear events.

Hide/Unhide

Filters the display for hidden/unhidden alarms.

Acknowledged

Filters the display to show only acknowledged alarms.

Purged

Filters the display to show orphaned alarms. An alarm is orphaned when its point has been deleted from the point configuration record. The purge is necessary since the point will never get an update for the alarm to be removed in a normal fashion.
Display Properties

Most Recent First

Use to retrieve the most recent alarms first. If you change the setting of this option, you will have to click the Get History button to refresh the view.

Max to Display

Use to set the maximum number of alarm records to show in the dialog box. The default is 100.
Buttons

Get History

Retrieves history for the point’s alarm conditions.

Get Next

If there are more records than what is displayed, Get Next retrieves the next quantity of a point’s alarm conditions.

Advanced

Provides options to change the point ID, set a date range, and to retrieve records from a different ELSALM.
