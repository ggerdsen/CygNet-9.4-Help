Alarm Reporting

Reporting of alarms to the Common Alarm Service (CAS) is independent of setting of an alarm bit in the Current Value Service (CVS).

Reporting to the CAS and GNS

To report an alarm to the CAS, you must enable the Report To CAS property for the point. To report an alarm to the General Notification Service (GNS) to send an alert about the alarm, you must enable the Report To GNS property for the point. These options apply to all alarms for the point and are part of the point configuration record in the Point Service (PNT).

Reporting Delay

Alarms and notifications can be delayed by configuring the Delay Reporting and Delay Value properties in the point configuration. Delayed reporting delays the CVS from sending an alarm record to the CAS and the GNS. The alarm bit is set immediately once the CVS determines that the value meets or exceeds the alarm setpoint; however, the alarm record will not be sent to the CAS or the GNS until the delay time is met.

You might specify a delay in reporting an alarm or notification for a short period of time to see if the point goes out of alarm. Once the specified delay period is exceeded, if the point is still in alarm, the record is sent. Setting a reporting delay is useful for eliminating nuisance alarms or to enable other processes to catch up before reporting an alarm, such as allowing a pump to reduce a tank level before reporting the tank level as an alarm.

Note: Delayed CAS alarms and GNS notifications records are persisted on shutdown and startup and replicated from a source to a destination. In the event of a failover these records will be available on the standby site after the failover is complete.

Reporting and Delay Configuration

The properties to enable alarm and notification reporting are configured in the point configuration in the PNT Editor:

Property Description

Report To CAS

Select to reporting point alarms to the Common Alarm Service (CAS). Options are 0 (No) or 1 (Yes). The individual alarm bits must be enabled.

Report To GNS

Select to report point alarms to the General Notification Service (GNS), which will send notifications when an alarm bit is set/reset. Options are 0 (No) or 1 (Yes). The individual alarm bits must be enabled.

GNS ID

Select the ID of the GNS record to use for the notification regarding the alarm. The record type must be "Event" or "Unspecified." Click … to select a GNS ID. See Configuring a GNS Record for information about setting up GNS IDs.

Delay Reporting

Select to delay reporting of the alarm to the CAS or to the GNS for some period of time to see if the point goes out of alarm. Options are 0 (No) or 1 (Yes). The individual alarm bits must be enabled.

Delay Value

Specify the amount of time to delay reporting alarms or processing notifications. The countdown begins when the alarm bit gets set. When set, that delay time governs all delay times for the point. Click … to enter the reporting delay value.

The format is as follows:

  • hh:mm:ss (for values up to 18 hours: 0-17 hours, 0-59 minutes, 0-59 seconds)
  • DD:hh:mm (for values up to 40 days: 0-39 days, 0-23 hours, 0-59 minutes)
  • Y:DDD:hh (for values up to 7 years: 0-6 years, 0-364 days, 0-23 hours)
