Hiding Alarms

Hiding an alarm is useful for hiding low-priority alarms. An alarm cannot be hidden if it is not acknowledged or if it is suppressed.

When you Hide an alarm, the service can be configured to prompt the user to enter a comment for hiding an alarm. The comment text is stored in the Comments property of the alarm. The configuring of the prompt for comment is done in the Alarm Service Advanced Configuration dialog box.

You can set up a filter to show hidden alarms.

Prompt Description

Do not prompt for comment

The Hide Alarms dialog box does not appear.

Prompt but don’t require comment

The Hide Alarms dialog box appears and a comment may be entered.

Prompt and require comment

The Hide Alarms dialog box appears and a comment must be entered. If a comment is not entered, a message appears saying that a comment is required.

To Configure Comment Options for Hide Alarm

  1. Right-click in the alarm view and select Alarm Auto Configuration.
  2. In the Alarm Auto Configuration dialog box, click Advanced.
  3. Use the Hide Alarm Comments Configuration option to select a user prompt, and click OK.
  4. Click Apply on the Alarm Auto Configuration dialog box.

To Hide an Alarm

  1. Click to select the alarm that you want to hide. Use Ctrl+Click to select non-contiguous alarms and Shift+Click to select contiguous alarms.
  2. Right-click and select Hide.
  3. In the Hide Alarms dialog box, type in a reason, if prompted, and click OK.
  4. Right-click in the alarm view and select Filter…. In the Alarm Filter dialog box, create a filter to hide the alarm. See Filtering Alarms.

To Unhide an Alarm

  1. To view any alarms that might be hidden, right-click in the alarm view and select Filter…. In the Alarm Filter dialog box, change the filter to unhide the alarm.
  2. Click to select the alarm that you want to unhide. Use Ctrl+Click to select non-contiguous alarms and Shift+Click to select contiguous alarms.
  3. Right-click and select Unhide.
