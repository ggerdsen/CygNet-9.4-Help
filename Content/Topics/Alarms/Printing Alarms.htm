Printing Alarms

You have a choice of sending alarm information to a line printer, printing the alarm view, or creating and printing an alarm report.

Printing the Alarm View

You can print all the current alarms in the alarm view by right-clicking and selecting Print Report. All the alarms and column headings in the alarm view will be printed. If you have hidden alarms that are not visible in the view, they will not print. If you want to print all of the alarms in the CAS, check the Filter option to see if any filters are set thus allowing only certain alarms to be displayed. It is recommended that you use a landscape orientation when printing.

To Print the Alarm View

  1. Right-click on the alarm view, and select Print Report. This prints the entire alarm view.
  2. Keep or change the print settings of the target printer.
  3. Click OK.

Printing the Alarm Report to a Line Printer

The CAS can be configured to feed alarms to a line printer. The alarm auto configuration file contains information regarding the printer port and speed.

To Print the Alarm Report

  1. Right-click in the alarm view and select Alarm Auto Configuration.
  2. In the Alarm Auto Configuration dialog box, click Advanced.
  3. Click Print Alarm Event and click on those alarm events that you want to format in a report and send to a line printer. (The alarm auto configuration file contains information regarding the printer port and speed.)
  4. Click OK.

Running a CygNet Alarm Activity Report

CygNet Reports contains an alarm report. The template for this report is named AlarmActivity.rpt and is stored in the SAMPLES group in the APPS. See CygNet Reports for more information about configuring and running the Alarm Activity report.
