Configuring the General Contract Monitor Device

Configuration of a GCM device mainly takes place in the DDS using the GCM device editor to set up where data comes from, what data you want to calculate, and how to retrieve it. This data is then mapped by point to the human-machine interface (HMI) component once you add and configure the CygNet GCM Nomination Grid control. The GCM device editor is made up of six property pages. Use these to configure your device.

Note: Before configuring the properties of your GCM device, ensure that you have met Group service and Facilities requirements.

Add a GCM Device

The following topics discuss how to add and configure a GCM device. A GCM device instance is defined as an Import/Export device type in the DDS.

To Add a GCM Device

  1. Using CygNet Explorer, open the Device Definition Service (DDS), and select the Import/Export Devices page.
  2. Right-click in the white space and select New Import/Export Device.
  3. In the Select a Device Typeâ€¦ list select General Contract Monitoring, and click OK.

    4. Note: If you right-click on an existing import/export device, that device is used as a template for your new device.

  1. Configure the device and save your work often by clicking Apply or OK (Apply saves the changes and keeps the dialog box open; OK saves the changes and closes the box).

Configure a GCM Device

The GCM device editor interface contains six configuration pages: Device, Settings, Group Filter, Calculations, UIS Commands, and Simulate. Each page is described in subsequent topics.

  • The Device page contains basic information about the device such as the device name, description, transaction logging, and device security. See Configuring the GCM Device.
  • The Settings page is used to configure advanced device settings, such as the group service, hierarchy and view, contract hour, subtractor inversion attribute, bit settings, daily nominations, and the commodity type. This page is also where you can save device settings to, or load device settings from an XML file. See Configuring GCM Settings.
  • The Group Filter page is used to define the location of each input within the navigation hierarchy. See Configuring Group Filters.
  • The Calculations page is where you add and configure calculations to be performed by the GCM calculation engine. See Configuring GCM Calculation.
  • The UIS Commands page is used to define the UIS commands used by the GCM system. See Configuring UIS Commands.
  • The Simulate page is where you can test your GCM setup in a simulated environment. See Simulating GCM Setup.
