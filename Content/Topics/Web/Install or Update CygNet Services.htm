Install or Update CygNet Services

The first step to installing the CygNet Web application is to make sure your CygNet services and support files are up to date. The Web server will pull required support files from your CygNet Application BLOB service (APPS), so the files on the targeted service must be up to date. The following table lists the files you will need. You will need additional files to run CygNet Studio files on the Web, which are part of your CygNet Studio or CygNet Vision Install.

If you have a large enterprise system with many remote locations, we recommend that you create a whole new set of services so that they may be managed together as one unit. If you have a smaller system, you may just want to create a dedicated GRP (Group Service), and HSS (Scripting Service).

To set up the new set of services download the most current CygNet SCADA software application from the Software Support portal on the Weatherford software website. Follow the install or upgrade procedure.

At a minimum you will need the following services:

  • RSM (Service Management)
  • ACS (Security Service)
  • PNT (Point Service)
  • CVS (UIS) (Real-time Service)
  • GRP (Group Service)
  • HSS (Scripting Service)
  • ELS (Log Service)
  • FAC (Facility Service)
  • TRS (Table Service)
  • VHS (Value History Service)
  • APPS (Application Service).
