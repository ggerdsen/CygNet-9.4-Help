Configure Application

The Configure Application option links to a form you can use to structure your CygNet Web application. You must have write authority on the custom/config.xml file below the root of your CygNet website to save any of the settings described below.

  • Site Settings
    • Application Level Information
    • General Settings for Real-time Data Screens
    • Overview Page Settings
    • Trend Page
    • Point List Page Settings
    • CygNet Studio/CygNet Vision Settings
    • History Report Settings
  • PageIDs
    • Default Page IDs
    • Page ID Attributes

Site Settings

Specify the Site Settings that dictate the structure and operation of the website. At any time you can click the Load Default button to load the suggested default values to the screen. The default values will not be saved until you click the Save Configuration button at the bottom of the screen. The options are described below:

Application Level Information

Option Description

Use Group Service Hierarchy

Specifies whether to use the Group Service Hierarchy. If set to No, the Group Service Hierarchy you will be limited to only CygNet Vision or CygNet Studio pages on the Web and many of the other site settings will disappear because they will not apply.

Group Service

Site and Service name of the CygNet Group Service that contains the hierarchical structure for your website.

Application BLOB Service

Site and Service name of the CygNet BLOB Service that contains the support files (mainly dlls) for your website.

Web Root Criteria

Specifies filter attributes to specify the Group Service node that will be the top of the hierarchy.

The default string is ~cat=R;~type=~W

Additional filter criteria may be added by using a semicolon as a separator.

For example: ;~desc=WebRoot for Areas/Routes

Use Replicated VHS

If set to True, the Web server will access history from your replicated VHS. The information regarding which VHS to use as the replicated VHS is stored in the CygNet UIS.

Logging

If set to True, Web application activity is logged to the <webroot>\custom\cygWebLog.txt file. The log file can be accessed through the View Log option in the administrative Web Console. See View Log for more information.

Notes:

Only turn on logging if you are troubleshooting, because it can slow down the system. Clear the log from time to time to help performance.

The AD group Authenticated Users must allow Modify and Write access to the <webroot>\custom\ folder for logging to take place. View Properties for the custom folder and set Permissions for Authenticated Users on the Security page.

Enable ActiveX Controls

If set to False, users will not be allowed to download or use ActiveX Controls. You will not be able to display Studio/Vision screens with ActiveX controls disabled.

Alternate Web Site

Specifies URL of alternate website that users are re-directed to if they are not authorized to access the CygNet WebControl site. This could be useful if you have both CygNet WebControl and WebView and you only want to publish one URL for all of your users.

General Settings for Real-time Data Screens

Option Description

Value Format

Specifies number format on data pages.

Update Frequency

Value (in msec) to set the update time period for real-time pages, such as Overview, Point List, and Summary.

Column Headings

Specifies the default column heading for UDC based pages such as Overview, Summary, Report, and Report History.

Overview Page Settings

Option Description

Show ALL Facilities on Overview

If set to True, all facilities will be displayed on a single page. If set to False, facilities will be displayed on multiple pages.

Warning: Loading all of the facilities at once can slow down the load time.

Trend Page

Option Description

Trend Date Span

Specifies the default time span Value (in days) for trending data.

Rollup Type

Specifies the rollup algorithm type for the Trend page.

Rollup Units

Specifies Rollup Units for use with thinning rollup type algorithms.

Point List Page Settings

Option Description

Point List Filter

Specifies string filter on System attribute to limit display on Point List page. Only points that have a System attribute value matching the Point List Filter will be displayed. If the setting is left empty or set to 0 or false, the Point List page will display all points for the selected facility.

CygNet Studio/CygNet Vision Settings

Option Description

Scrollbar Override

Select from the following scroll bar options that will be used for the Studio screen from within the Web:

  • No Override — use the scroll bar setting from the Studio screen
  • No — never use scroll bars on the Studio screen
  • Yes — always use scroll bars on the Studio screen
  • Scroll bars with Flex — resize the screen with the Web browser to just before the point where the screen becomes difficult to read.

History Report Settings

Option Description

History Report Date Span

Specifies default date span for history report.

Page IDs

PageIDs are used by the Group Service to organize the pages that are available from each level of the Web hierarchy.

List the Page IDs for pages you will use on your website. You may add custom pages, change the names that appear on the pages, and specify start pages for the Studio screens to be displayed on the Web.

The default page IDs and corresponding page descriptions are listed below:

Page ID Attributes

Page ID Attribute Description

See defaults options in table above or use any character combination of up to 4 characters.

Tab Desc

Specifies the name that will appear on the page of the website.

Page

cygStudio.asp

Start Screen

(for Studio pages ONLY)
Specifies the CygNet Studio screen to load one of the following.

  • A BLOB reference (\\sitename.apps\BLOBApp\screenname.csf)
  • A Microsoft Windows file reference (c:\folder\subFolder\screenname.csf)

To Add New Page ID

  1. Click the Add Page ID button, an empty row will appear.
  2. Enter ID, Tab Desc, Page, and Start Screen (if applicable).
  3. When finished click the Save Configuration button.

To edit the attributes of an existing PageID click and type in the attribute boxes to replace the previous settings.
