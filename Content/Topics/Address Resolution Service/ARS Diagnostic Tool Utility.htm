ARS Diagnostic Tool Utility

The ARS Diagnostic Tool utility provides administrative statistics on Address Resolution Services (ARS) for advanced administrators, integrators, and CygNet support. Available diagnostic information includes service validation, iterator test, site consistency, cycle times, and CVS metadata file synchronization. The ARS Diagnostic Tool utility allows the addition of other domains to monitor time synchronization across multiple domains in a redundant environment.

The ARS Diagnostic Tool utility file is stored in the CygNet\Utilities directory (ArsDiags.exe) on the host server.

Starting the ARS Diagnostic Tool

Installing and Launching

To install and launch (start) the application or utility using version management:

  • Use CygNet Client Installer to install and launch the application. Once installed you can use CygNet Client Installer to start the application.

-OR-

  • If previously installed, click Start > All Programs > CygNet > <name of the application or utility> or double-click the desktop shortcut.

Alternatively, browse to the CygNet\Utilities directory using Windows Explorer and double-click the program icon. You can also start an application or utility from a command line. These methods do not provide version management.

User Interface

The ARS Diagnostic Tool utility has five views:

  • Validation — performs validation for the master ARS. This is the default view.
  • Consistency Monitor — compares ARS services for consistency within a domain.
  • Time Test — displays ARS information and related Cycle Time for selected domains.
  • File Synchronization — displays data associated with the file selected in the Current File menu.
  • Iterator Test — performs an iterator test for the selected ARS.

To select a view, click the desired name from the list on the left side of the page. The default view in the utility is Validation.

Validation

ARS validation is performed on the Validation view. When you click on Validation in the list on the left side of the page, a blank Validation page opens. Click Do Validation to start validation. Once validation is complete, the Validation pane will list the results. Scroll through the result list to view and correct any anomalies and exceptions before proceeding with other ArsDiags tasks.

Time Test

The Time Test page displays ARS information and related Cycle Time for selected domains, allowing you to monitor time synchronization across multiple domains in a redundant environment.

For each domain the Cycle Time, Earliest, Latest, and Delta values are displayed. When more than one domain is displayed, a Summary section displays the Cycle Time, Earliest, Latest, and Delta values across all domains.

Other columns displayed on this page include: the executable version of the ARS, the time the executable started, the port on which it is running, the name of the computer where the executable is running, whether it is the master ARS, the number of services running under the ARS, the activation status, the remote time of the service, the time delta in milliseconds, the number of active messages, and the CygNet message count since the service started.
