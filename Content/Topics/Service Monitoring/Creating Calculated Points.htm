Creating Calculated Points

SVCMON has a built-in set of algorithms that can be applied to SVCMON UDCs for performing calculations. These include:

  • Delta (percentage change between update cycles)
  • Rate of change (in milliseconds, seconds, minutes, hours, days)
  • Scaling (in bytes, kilobytes, megabytes, gigabytes).

Calculated points are based on source points. Multiple calculated points can be created for each source point.

Not all calculation types are appropriate for all source points. For example, it would make sense to monitor the rate of change for the Value History Service database size, but it wouldn’t make sense to monitor the rate of change for the operating system version.

The Calculated Points view is used to add calculated points. The process of adding a calculated point involves:

  1. Adding a point for the source UDC (the source point on which the calculation is based). This process is described in Creating SVCMON Points. A calculated point can only be created if a source SVCMON point exists.
  2. Defining the calculated UDC(s). This process takes the base SVCMON UDC and creates a new UDC based on the selected algorithm. Calculated UDCs have associated tag templates just as other SVCMON UDCs. Multiple calculated UDCs can be created based on the same base UDC.
  3. Creating the calculated point(s) for the source points.

Calculated UDCs are stored in the Table Reference Service in the ~UDCCALC category.

Once a calculated point has been added, it will be listed in the Explore view.

As with SVCMON points that monitor info items, calculated points utilize certain Point Service properties. These properties are assigned values by SVCMON and should not be changed. These include the Facility and UDC properties on the General page, the General Data Field 1 (source UDC) and Table Field 1 (calculation type) on the Application page, and the Point Reference properties on the Point Reference page.
