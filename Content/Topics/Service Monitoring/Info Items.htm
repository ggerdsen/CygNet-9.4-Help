Info Items

An Info Item is a piece of information about the system. There are many info items available in the system ranging from host server machine statistics, such as memory and disk space usage, to service configuration statistics, such as General Notification Service (GNS) pending notifications or the Value History Service (VHS) value file size. The Service Details View in CygNet Explorer displays service info items and their values. Info items also can be viewed and trended with the CygNet Service Diagnostics utility.

Info items are the SVCMON tags. SVCMON tags are based on a set of SVCMON-specific system UDCs and CygNet services or sites. Each UDC has an associated template. As point tags are created for a UDC, the settings of the template are applied to the point tag. A template can be changed, and the changes applied to all point tags that use the UDC. All SVCMON UDCs begin with the letters SVM.

Most Info Items have a corresponding SVCMON UDC. By creating points for these SVCMON UDCs you can easily monitor Info Item values and utilize point features such as alarming or reporting to history (VHS). To create SVCMON points for info items, use the CygNet ServiceMon Administration utility.

Info Item values can be retrieved via script using the GetGenserveInfo global function.

Some info items can be sent to a service using a Sent Info Keyword Task in the Master Scheduling Service (MSS) to trigger the service to perform an action such as an on-demand backup of the serviceâ€™s files or the creation of a new history journal file.

Info Item Details

The following info items are common to all services. They are organized by the pages in the Service Details view in CygNet Explorer.

  • Host Details
  • Network
  • Process Performance
  • Machine Performance
  • Service Generic

These info items are service specific:

  • Service Specific
  • Replication

These special client side items are available for DBS-based services only.
