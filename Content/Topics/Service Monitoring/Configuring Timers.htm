Configuring Timers

Timers determine the update rates for SVCMON points. The Timer Config view is used to configure timers and to assign SVCMON points to timers. The maximum update rate is 1 second. The minimum update rate is 24 hours (86,400 seconds). Any number of timers can be configured.

Grouping points into timers can improve service performance. Some point types, such as memory usage, change more frequently than operating system version. For example, SVCMON points that gather data from remote locations can be polled less often than those locally. In addition, separating points into timer groups based on point types also makes sense.

A default timer is created by the system named DEFAULT_SVMTIMER. All SVCMON points are initially assigned to this timer, which has an update rate of 60 seconds.

The Time Config view displays all configured timers grouped by Timer. Each column can be filtered using the filter row at the top of the data grid to narrow the selection.

  • Click to select the desired column text box, then type the desired filter; or click … to view the Filter Column by Text dialog box.
  • For the Timer column, you can filter by configured timers.
  • The default filter for all columns is <all>.
