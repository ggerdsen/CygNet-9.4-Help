SVCMON Terminology

There are a number of terms specific to the SVCMON and its Administration utility. A basic understanding of these terms will help you when working with Service Monitoring applications in CygNet.

  • Info Item
  • Point Templates
  • Timers
  • Calculated Points

Info Item

An Info Item is a piece of information about the system. There are many info items available in the system ranging from host server machine statistics, such as memory and disk space usage, to service configuration statistics, such as General Notification Service (GNS) pending notifications or the Value History Service (VHS) value file size. The Service Details View in CygNet Explorer displays info items and their values.

Most info items have a corresponding SVCMON UDC. By creating points for these SVCMON UDCs you can easily monitor Info Item values and utilize point features such as alarming or reporting to history (VHS). All SVCMON UDCs begin with the letters SVM.

Some of these SVCMON UDCs that represent info items are specific to sites, while others are specific to service types. Most are applicable to all service types. For example, the "site status" UDC (SVMSTST) is applicable only to CygNet sites. This UDC monitors the health of a site as a whole. The "value file percentage full" UDC (SVMVHSVFPF) is applicable only to the Value History Service. It monitors the percentage full of the disk space allocated to the VHS. The "total CPU usage" UDC (SVMTCPUU) can be applied to any service. It monitors the CPU usage of the host server.

See Info Items for a list of available info items and their corresponding SVCMON UDCs.

Point Templates

Another type of SVCMON tag is a point template. A point template is a UDC-specific point stored in the Point Service. All point templates have a Facility of ~TEMPLATE and the UDC is the SVCMON UDC (for example, SVMTCPUU which represents total CPU usage).

Point templates can be applied to any SVCMON point that uses the same UDC as the point template. Point templates can be applied to SVCMON points that monitor info items, timers, and calculated SVCMON points. They can be applied at the time the points are created or anytime thereafter, even if the tags were not initially based on the template.

Note that the Status property of a point template (in the Point Service) is set to Inactive. As such, the tag is listed in the PNT but not SVCMON. There is no need to change this setting.

Timers

Timers define the update rates for SVCMON points. A timer has one interval, and all points added to that timer are polled at that interval. Any number of timers can be defined, and you can associate any SVCMON point with any timer. The maximum update rate is 1 second.

The CygNet ServiceMon Administration utility creates a default timer which has an update rate of 60 seconds. All new SVCMON points are automatically assigned to this timer. Points can be reassigned to other timers, and the update rate of the default timer can be changed.

Calculated Points

Calculated points are based on the raw values returned by standard SVCMON points. Calculated points can represent rates, deltas, or scaled value of the raw value. Multiple calculated points can be based on the same raw value.

SVCMON calculated points are not HyperPoint Scripting Service points. The algorithms are defined in SVCMON.
