CygNet Service Diagnostics Utility

The CygNet Service Diagnostics utility provides access to real-time service, site, and machine statistics. Data presented in the Service Diagnostics data grid can be filtered, sorted, printed, exported to a CSV file, refreshed at regular intervals or on demand, and trended. You can view service information and log settings for any service. You can also save configuration settings to an .xml file, import settings from or export settings to an XML file, and settings can be set to persist from utility instance to instance.

By default the data grid always displays the status of the connected services and the time it takes to ping the service in milliseconds (ms). Possible status values are OK, OOS (out of service), Failed, or Defined.

See Info Items for more information.

The CygNet Service Diagnostics utility is stored in the CygNet\Utilities directory (SvcDiags.exe) on the host server.

Starting Service Diagnostics

Installing and Launching

To install and launch (start) the application or utility using version management:

  • Use CygNet Client Installer to install and launch the application. Once installed you can use CygNet Client Installer to start the application.

-OR-

  • If previously installed, click Start > All Programs > CygNet > <name of the application or utility> or double-click the desktop shortcut.

Alternatively, browse to the CygNet\Utilities directory using Windows Explorer and double-click the program icon. You can also start an application or utility from a command line. These methods do not provide version management.

When you open the CygNet Service Diagnostics utility the following dialog box appears; it can be customized to show user-selected service info items.

User Interface

The CygNet Service Diagnostics utility consists of a menu bar, a toolbar, a display grid, and a status bar. The following tables list the options accessible using the menus, toolbar, and context menu.

Menus

Three tabs are available on the menu bar to access various service diagnostics functionality, via the File, View, and Help menus.

File Menu

The File menu provides access to the following options.

Option Description Shortcut Tool

Open File

Ctrl+O

 

Save As

Saves current settings to a new Service Diagnostics configuration file

Ctrl+S

Export Data

Exports info items data to a CSV file. Specify a file name and path and click Save. You cannot export while the utility is running

    

Print

Prints info items data. You cannot print while the utility is running. Specify a printer and click OK

    

Recent Files

Lists all recently opened Service Diagnostics configuration files

Help Menu

The Help menu provides access to the following options.

Option Description Shortcut Tool

About CygNet Service Diagnostics

Click to open the About CygNet Service Diagnostics dialog box to view application information.

Available information includes the following:

  • The application name, version, and release date
  • Copyright information
  • The CygNet Software website address, including a hyperlink to the website
  • The log file location, including a hyperlink to the log file
  • The location of the application's file executable

In addition to information about the application, the About box includes the following buttons:

  • Client Settings — opens the Client Settings dialog box to view and modify client log settings for the application on an ad hoc basis. See Client Log Settings in the Logging section for more information.
  • Diagnostics — access the following options for additional analysis from the drop-down menu:
  • Loaded Modules — use this option to view and search file information for modules loaded in your system. Double-click any result row to view properties for that module. See Loaded Modules for more information.
  • Gather Logs — accesses the Windows file directory where the open log files are located. Double-click to select and view an application log file.
  • User Local Crash Dumps — Local crash dumps are not enabled. See Generating Crash Dumps for more information.
  • Close — click to close the About box.
