CygNet Explorer User Interface – Context Menus

The two panes in CygNet Explorer access different context menu options: the service hierarchy (left pane) and the service pane (right pane).

Service Hierarchy

The context menu available for the services in the service hierarchy includes the following options:

Option Description

Filter

Filter services displayed in the service hierarchy.

Options include:

  • New
  • Edit
  • Delete
  • Apply

Show

Change the way the name of the services are displayed in the service hierarchy.

Options include:

  • Service Only — for example, CAS
  • Description Only — for example, Common Alarm Service
  • Description (Service) — for example, Common Alarm Service (CAS)
  • Service (Description) — for example, CAS (Common Alarm Service)

Note: The services will be reordered alphabetically.

Auto Refresh

Click to automatically refresh status indicators when services start or stop.

Note: Only seen for the top-level domain node of the service hierarchy.

Service Control

Manage the service as necessary. The service icons and states will automatically refresh as the services start, stop and restart. Options include:

  • Start — The RSM will start the selected service. See Service Management for more information.
  • Stop — The RSM will stop the selected service. See Service Management for more information.
  • Kill — Kill the selected service. When a service is killed, a Microsoft Windows "stop" message is sent to the service. Only use if the service is not responding to a normal CygNet stop request.See Remote Service Manager and Service Management for more information.
  • Restart — The RSM will restart the selected service. See Service Management for more information.

Notes:

These controls are disabled for the primary RSM.

If you have multiple RSMs configured, the system will automatically detect which services to control.

Launch Console

Launch the CygNet Console for the selected service. Opening CygNet Console this way utilizes version management. If User Account Control (UAC) is enabled, launching CygNet Console will require administrator-level permission, if CygNet Console requires updating. See CygNet and User Account Control for more information.

Note: You can ensure the launched application runs on the same (or different) domain by setting the /domain parameter in the .app file.

Service Details

View CygNet Service Information (Service Details) for the selected service. See Viewing Service Details for more information.

Remote Desktop

Connect to a remote computer running the selected service.

Properties

Show or change the properties for the selected item. The dialog box displayed will depend on the item selected:

  • If you have selected a service in the service hierarchy, the SITE.SERVICE Properties dialog box will display.
  • If you have selected a record in the service pane, the appropriate dialog box will open for the selected service.

MSS

Execute Now

Select this option to execute an MSS task immediately. For one instance only, Execute Now issues the selected task to its respective service. The task's priority determines when the task will initiate an actual poll. After the task executes, polling returns to the task's scheduled routine.

Note: You must either have administrative access to the service or have "update" level access to the security event EXECNOW to run this task.

Activate

Select this option to activate a highlighted task without opening its property sheet.

Deactivate

Select this option to deactivate a highlighted task without opening its property sheet.
