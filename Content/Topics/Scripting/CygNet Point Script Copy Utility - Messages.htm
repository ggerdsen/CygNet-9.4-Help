CygNet Point Script Copy - Messages

The following table describes messages which may appear when using the CygNet Point Script Copy utility.

Message

 Description

Select a valid script source file.

The file specified in the From File property does not contain script source. Check the file name and path.

Select a script for the source.

The From File option requires you to select which script to copy. Select the script using the Script box under the From File option.

No source script selected in file

A script name is missing for the copy from action. Under the From File in the Script box, select the Script title from the list.

No update necessary. Scripts are the same.

An update did not occur because the scripts were identical.

The File 'file-name.txt' exists! Do you want to Overwrite or Append to it?

Message is displayed when you select a scripted point to be copied to a script file that already exists. If you select Append, the script is included at the end of the file.

Found n Group Rollup Instances

The n represents the number of group rollup instances found during the script copy. If 0, the service had no group rollup points.

ERROR — Could not open tag file

The Points in File text file could not be found. Verify the file name and path.

Reading in n tags from file file-name

Indicates the number of scripted tags in the source file.

Failed to read bytes from %s

Indicates that the values could not be accurately read. Make any changes to the script, and try again.

Could not read source point template

This occurs when a scripted point is copied from a different site. Check the default service from where the script is copied from.

All Done

Processing is complete. This message is shown both for Test Copy and Perform Copy actions.
