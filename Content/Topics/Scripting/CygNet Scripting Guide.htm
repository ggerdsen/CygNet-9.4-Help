CygNet Scripting Guide

Scripting provides a method to extend your CygNet installation to meet specific requirements, whether analyzing large amounts of data or designing a custom interface. CygNet includes an integrated set of automation libraries that facilitate the creation of simple to complex logic.

CygNet scripting environments use Microsoft Visual Basic for Scripting (VBScript). The VBScript Primer topic provides information about the aspects of VBScript most relevant to scripting in this setting, however it is not intended as a programming tutorial; it assumes a level of familiarity with the principles and structure of VB, VBA, or VBScript. Refer to the online MSDN VBScript article for more complete information.

To familiarize yourself with scripting, it is recommended to use a working CygNet system, create some test points (perhaps simulated), and experiment with the example scripts provided. It is also helpful to refer to the various Scripting Reference Libraries which explain each object and function in detail.

The following topics describe the basics of writing and using scripting in a CygNet environment. Multiple scripting examples are provided.
