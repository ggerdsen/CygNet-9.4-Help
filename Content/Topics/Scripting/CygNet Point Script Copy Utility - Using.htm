Using the CygNet Point Script Copy Utility

Whenever script copy is performed using this utility, a log is written to the directory from which the utility was run. The file name is PointCfgScriptCopy001.log. The file contains the results of the copy.

CygNet Software recommends that you always use the Test Copy before using the Perform Copy option to verify there will be no errors. There is NO "undo" for the Perform Copy.

Note: CygNet also provides a set of script library files that can be installed using the CygNet Point Script Copy utility. See Script Library Files for more information.

Service Properties

Service Property Description

PointCfg Service

The PNT in which the scripted points reside or will be written to.

Target HSS Service

The current value service of the scripted points. This property filters the source when selecting points and when copying scripts to points in a service. The services listed as those CVS’s for which scripting has been enabled. CygNet Software recommends that only the HyperPoint Scripting Service (HSS) be used for point scripting.

Source Options

Script Copy Source (Copy From) Description

From All Points

Copies the scripts from all scripted points in the selected PointCfg Service and the selected Target HSS.

From Point

Copies the script from the selected point. Use … to select the point.

From File

Copies scripts from a file. A script file may contain more than one script. If this is the case, the script names are listed in the Scripts box. You can choose all scripts or a particular script.

Target Options

Script Copy Target (Copy to) Description

To Points in File

When this option is selected, the source script is copied to the points listed in a file. The file is a text file that lists each point identifier on a single line. A full point identifier must be included. Any valid CygNet tag string format can be used

(SITE.SERVICE.POINTID;

SITE.SERVICE:LONGID; SITE.SERVICE.POINTID:LONGID; SITE.SERVICE::FACILITY.UDC)

Example file listing:

MYSITE.HSS::MESA.AVEFLOW

MYSITE.HSS::AZTEC.AVEFLOW

MYSITE.HSS:XY7021_AVEFLOW

MYSITE.HSS.00002343

If a point listed in the file does not exist, it will be created. If the point does exist, it will be updated only if its HyperPoint property is enabled.

You can use this option with the From Point or From File source options.

To Point in Service

When this option is selected, the source is copied to a HyperPoint in the system. Use … to select the point.

Note: The target point must be a HyperPoint (that is, have its HyperPoint property enabled) or the script will not be written to the point.

You can use this option with the From Point or From File source options.

To Script File

When this option is selected, the source script is copied to a file. If the target file exists and you want to overwrite its contents, click to enable the Overwrite option. If this is not enabled, the existing target file will be appended.

You can use this option with any of the source options.

To Default Service Point in Source

When this option is selected, the script is copied to the point listed in the source. Generally, this option is used with the From File source option. As such, the script would be copied to the point identified in the script file.

To Group Rollup Instances

This option applies when you have group rollup points (other than the template points). The source script is copied to existing group rollup point(s).
