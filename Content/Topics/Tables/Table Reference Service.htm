Table Reference Service

The Table Reference Service (TRS) stores selections for table-driven fields and Uniform Data Codes (UDC). Some table fields are system-defined. Others are user-defined.

Using the TRS Editor

Table and table entry configuration is made in the TRS Editor, or the Properties for Entry <entry> in Table <table> dialog box.

  • Right-click in the service pane and select New to configure a new table record.
  • Right-click a table entry and select Properties (or double-click) to open a table record. By default the TRS Editor opens in View (or read-only) mode. Use the Previous and Next buttons to scroll through the configured entries in the TRS in View mode.
  • Click Edit to modify a TRS entry. The Previous and Next buttons are disabled in Edit mode.
  • If changes are made to a table entry and the TRS Editor is canceled, a prompt dialog will display allowing the user to Save, Discard, or Cancel and return to the editor.
  • Click Audit History to view an audit history trail for the table entry. See Auditing for more information about how auditing works in CygNet Software.
