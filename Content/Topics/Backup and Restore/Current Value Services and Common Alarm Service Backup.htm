Current Value Services and Common Alarm Service Backup

This topic describes automatic backup for CygNet’s Current Value Services and the Common Alarm Service. The CVS services include:

  • HSS
  • OPCIS
  • SVCMON
  • UIS

The automatic backup of the Current Value Services and Common Alarm Service is limited only to the service configuration file and the persisted data files. Backup of data files is not an option due to the fact that the points served by the Current Value Service or Common Alarm Service will be written to the service when the CVS is restarted. Only persisted data is written to the service when it is started if the point/service is configured to persist values.

The service configuration file (.cfg) and the persisted data files (CVS.Persist.bak, CVS.Persist.dat) are backed up to the Backup location specified in the BACKUP_PATH keyword for each CVS.

The service configuration file (.cfg) and the persisted data files (AlmHistPersist.bak, AlmHistPersist.dat, CasMetadata.dat, CasPersist.bak, CasPersist.dat) are backed up to the Backup location specified in the BACKUP_PATH keyword for the CAS.

Backup Keywords

Configure the following keywords with a valid parameter in each service configuration file to enable automatic backup of the service configuration file or persisted data files for the selected CVS or CAS:

  • BACKUP_PATH
  • BACKUP_TIME
  • COPY_BACKUP_TO

See Backup Keywords and Paths for definitions. See the relevant Service Configuration File Keyword document (CAS, HSS, OPCIS, SVCMON, and UIS) for configuration keyword defaults and options.
