Address Resolution Service Backup

This topic describes automatic backup for CygNet’s Address Resolution Service.

Although ARSs are redundant in a multi-host system, enabling backup is recommended in case of file corruption or to preserve any custom settings in associated files such as the configuration file. If backup of the ARS is enabled, the following files are backed up to the Backup location specified in the BACKUP_PATH keyword for the ARS:

  • service configuration file (Ars.cfg)
  • CVS metadata file (CvsMetadata.xml)
  • license file (CygNet.lic)
  • domain definition files (CygNetDomainDefs.xml)
  • meter data files (CygNetMeter.db)
  • time zone data (CygNetTimeZones.xml)
  • service definition data files (ServiceDefs.[domain].db)

Backup Keywords

Configure the following keywords with a valid parameter in the ARS configuration file to enable automatic backup of the above files:

  • BACKUP_PATH
  • BACKUP_TIME
  • COPY_BACKUP_TO

See Backup Keywords and Paths for definitions. See ARS Service Configuration Keyword for configuration keyword defaults and options.
