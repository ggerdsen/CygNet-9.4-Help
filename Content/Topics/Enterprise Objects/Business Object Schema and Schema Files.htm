Schema and Schema Files

The Business Object Schemas define the XML data types supporting all configured Business Object Interfaces, allowing CygNet to provide external systems with data in terms of the Enterprise data model. These XML schema definition files (.xsd) are dynamically created, based on the Business Object Package definition, and will be updated as Business Object Interface configuration changes occur.

Note: The CygNet Business Object schemas stand alone, and do not reference any CygNet Native Object schemas.

Schema Generation

To Generate an XSD File

  1. On the Package menu, click Open to navigate to the name of the desired package file.
  2. Click the package file name to select it, then click Open.
  3. Click Info to display package and file information, and verify the package selection.
  4. Click Generate XSD. You will be prompted to select a directory in which to place the XSD file.
  5. To choose an existing directory location, navigate to the desired location and click to select it.
  6. To create a new directory location, click Make New Folder. Type in a folder name and click to select the new folder.
  7. Click OK.
  8. Your XSD will be generated and will be placed in the selected directory. A confirmation message will be displayed.
  9. Click OK.
