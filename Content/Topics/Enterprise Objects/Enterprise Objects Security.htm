Enterprise Objects Security

Security for the Enterprise Integration Suite is administered by the Access Control Service (ACS).

See Security for more information about general CygNet security.

Some components of Enterprise Objects, including the Enterprise Operations EIE, and the Enterprise Gateway, involve additional levels of security. Security permissions are verified by the ACS.

The required security level for Enterprise Objects operations is dependent on the particular operation being executed, and on the service it interfaces with. To execute any Enterprise Objects operation, you must have security access at or above the level required for all processes required in the execution of that specific operation.

Example

If the operation "GetFacilityPointList" is configured with the security Application/Event "ENTOPS/ACCESS" the mimimum security level required in ENTOPS/ACCESS to execute the operation is "1-Read" since it is a "read" operation. Similarly, since "AddFacilities" is an "add" operation, it would require security level "3-Add," and so on.
