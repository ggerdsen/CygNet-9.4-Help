Troubleshooting Procedures

To Install CygNet Business Object Builder

CygNet Business Object Builder requires Microsoft .NET Framework for installation, in accordance with the CygNet System Requirements document. Contact your IT administrator for assistance installing the .NET Framework, and to ensure adherence to the prerequisites listed in the CygNet System Requirements document.

Attempting to install the Business Object Builder without proper prerequisites will generate an error, and installation will fail.

Installation failure using CygNet Client Installer produces the following error message.

Installation failure using CygNet Business Object Builder Installation produces the following error message.

See Business Object Builder Installation for CygNet Business Object Builder installation procedures, once system requirements are met.

To Reconfigure the ODBC Maximum Query Size

  1. In the CygNet\Support\ODBC directory, open the OASQL.INI file.
  2. Under the [Common] heading, add the keyword/value pair "QueryMaxSize=xxxx" where xxxx is at least as large as the size indicated in the response file error message.
  3. Restart the UIS.
