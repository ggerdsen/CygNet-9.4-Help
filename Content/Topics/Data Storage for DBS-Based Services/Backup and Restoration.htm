Backup and Restoration

ESE can make consistent copies of CygNet data while ESE is online and actively modifying the data state. Databases can continue to be queried and updated while the backup is being made. In CygNet there are two methods for performing a database backup:

  1. Enabling and configuring the appropriate keywords in the DBS-based service’s configuration file:
    • BACKUP_PATH
    • BACKUP_TIME
    • COPY_BACKUP_TO
  1. Schedule a backup using the MSS.

The following topics provide additional information about backups for the CygNet DBS-based services:

  • See Backup and Restore for complete instructions on planning and implementing database backups and database restoration.
  • See DBS-Based Services Backup for information about automatic backup, backup keywords, and full and incremental backups.
  • See Transaction Log Files and DBS Backup for information about how transaction log files are handled during a full and incremental backup.
  • See DBS Restore Backup Utility for information about using this utility to restore a database backup file from a DBS-based service.
