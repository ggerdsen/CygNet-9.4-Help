Emerson ROCPlus EIE

Note: The Fisher EIE has been updated and its functionality has been split by native protocol between the Emerson ROC EIE and the Emerson ROCPlus EIE. If you currently have Fisher EIEs deployed, refer to Fisher EIE documentation released in earlier versions of the CygNet Help. If you are installing Fisher or Emerson field devices and want to use EIEs to support their use with CygNet and/or you have upgraded to Emerson ROC EIEs or Emerson ROCPlus EIEs, refer to Emerson ROC EIE and Emerson ROCPlus EIE documentation.

The Emerson ROCPlus EIE supports interoperation with several common Emerson Process Management field devices, including ROC 809, 827, and 800L devices. Non-liquid-meter (orifice and turbine) remote devices can use either Series 1 or Series 2 hardware; liquid meter devices can only use Series 2 hardware.

Native data groups available for this EIE reflect TLP (point type, location/logic, parameter) groupings offered by Emerson ROCPlus field devices. In such cases, opcodes and TLPs are hard-coded into the CygNet data group. This streamlines access to common Emerson ROCPlus TLP groupings, enabling read/write messaging to occur with a minimum of setup.

In addition to native, predefined data groups, CygNet provides a special data group type called a "Generic TLP Data" data group. You can add as many instances of this data group type as necessary to form your own collections of up to 30 TLPs per "Generic TLP Data" data group. Some might be TLPs already part of a native data group and some might not be predefined by CygNet at all. Effectively, you can use the "Generic TLP Data" data group type to access any TLP on any supported Emerson ROCPlus model and to group and use them according to your needs.

Although the Emerson ROCPlus EIE depends on various model-specific device template files, Emerson ROCPlus device template files are relatively inflexible. Each was written to interface with a proprietary (ROCPlus) protocol and so they must meet Emerson ROCPlus specifications very closely.

CygNet supports Emerson field devices that use either the ROC protocol or ROCPlus protocol, depending on model number. For information about Emerson ROC devices, see Emerson ROC EIE.

Note: A valid "ROC Config" (RtuConfig) data group must be retrieved before other data groups can be retrieved or sent. The data group should always be repolled if there are configuration changes to the field device, after a CygNet upgrade, after importing a device or after modifying the device template file.
