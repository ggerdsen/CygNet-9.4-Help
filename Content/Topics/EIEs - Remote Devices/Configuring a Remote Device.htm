Configuring a Remote Device

The following procedures are generalized workflows for configuring a remote device. Additional configuration considerations are certain. For instance, you might want to configure your remote device to interface with electronic flow measurement (EFM) devices and data. Or you might have other specific complexities beyond the scope of this topic.

To Add a New, Blank Remote Device

The following steps explain how to add a completely new remote device from a list of installed EIE drivers.

  1. Using CygNet Explorer, open the Device Definition Service (DDS), and select the Remote Devices page.
  2. Right-click in the white space of Remote Devices page, and select New Remote Device.
  3. Select a device type from the Select a Device Type... list and click OK.
  4. Configure the properties for the remote device and click OK.

To Add a New, Pre-Filled Remote Device

The following steps explain how to add a partially copied ("shallow") remote device from the existing Remote Devices.

  1. Using CygNet Explorer, open the Device Definition Service (DDS), and select the Remote Devices page.
  2. Right-click on an existing remote device in the Remote Devices.
  3. Click New. A partially configured ("shallow") copy of the existing remote device appears.
  4. Configure the unfilled properties for the remote device and click OK.

For information about copying remote devices, see Copying a Remote Device.

To Configure a Remote Device

  1. Using CygNet Explorer, open the Device Definition Service (DDS), and select the Remote Devices page.
  2. Double-click the remote device you want to configure.
  3. Click the Facilities page to link facilities to the device. The default facility is the device facility.
    1. Double-click the device facility to show its properties and enter attribute data. To change its description, you must change the description of the device on the General page. Best practice is to have the default facility represent device data (board temperature, voltage, communication statistics, and so forth).
    2. Click Add Link to add additional facilities. The additional facilities must already be set up in the Facility Service (FAC). Best practice is to have the linked facilities represent operational data. Repeat as necessary for the required number of operational facilities.
    3. Click Apply.
  4. Click the Data Groups page to add data groups and map UDCs. Default data groups are pre-added.
    1. To add more data groups, including ordinalized data groups, click Add.
    2. Select the desired data group and click OK.
    3. Fill in data group properties according to your needs and click OK.
    4. Select the data group and click Map UDCs. This enables you to map CygNet uniform data codes (UDCs) to device data group elements.
    5. Select a data group element and click Map.
    6. Select the correct UDC and click Assign UDC. Use the UDC Category drop-down filter to narrow possibilities. Click OK.
    7. To remove a UDC assignment, select a data group element and click Remove Mapping, then click OK.
    8. Click Apply.
  5. Click the UIS Commands page to define commands for the device. Default commands are pre-added.
    1. To add more UIS commands, click Add.
    2. Type a command Name and Description.
    3. Add command components.
    4. Assign a command component to a data group using the Choose Group button.
    5. Define additional component parameters as needed.
    6. Click OK.
  6. When you are finished with all configuration changes, click OK to commit changes to the DDS.
