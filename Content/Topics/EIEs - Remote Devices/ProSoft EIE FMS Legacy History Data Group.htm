FMS Legacy History Data Group

The "FMS Legacy History Data" (GmrHist) data group is required by measurement to collect historical data. The "FMS Legacy History" data group does not retrieve data directly from a remote device; instead, it depends on another data group to supply data to it.

The "FMS Legacy History" data group is defined once in a device template file, but it can be instantiated in the remote device editor and ordinalized to scale to your measurement history collection needs. You must associate each such "FMS Legacy History" data group with a matching support data group. For example, if you instantiate "FMS Legacy History" ordinal 1, you must also instantiate support data group ordinal 1.

Support data groups are assigned to an "FMS Legacy History" data group in the supportDg element of your device template file. "FMS Legacy History" data group elements are assigned to FMS Legacy History Data Items in the histItems element of gmrMaps element in a device template file.

The TmOrient data group element does not appear in a device template file, but it commonly appears as part of a DDS transaction viewed in the CygNet DDS Viewer utility (DdsViewer.exe). This element indicates whether a device supplies history in leading or trailing time. If its value is 1, the history record uses leading time. If its value is 2, the history record uses trailing time.

Example

<GmrHist niceName="FMS Legacy History Data">

<dgElements/>

<supportDg>

<HistDaily required="true"/>

</supportDg>

<uccRecvParms>

<SDate desc="Start Date" required="true"/>

<EDate desc="End Date" required="true"/>

<MeterId desc="Meter ID Override" required="false"/>

</uccRecvParms>

</GmrHist>

To Create a Data Group

  1. Using a text editor or XML editor, open the device template file to which you want to add data groups.
  2. Define a new data group.
    1. Define a dgElements element.
    2. Define a supportDg element.
    3. Define all support data groups from which you require data. Use only one history data group.
    4. Define a uccRecvParms element in the data group.
    5. Define one or more RecParmName elements.
  3. Define any other required data groups.
  4. Validate and load your template.
