Data Groups

With few exceptions, data groups must be defined in a device template file in order for them to be available for use on a remote device. Which data groups are defined by a device template file depends on protocol, device type, and unique configuration.

CygNet distributes sample device template files for its EIEs, each of which typically serves one or more hardware models along with applicable firmware. Therefore, the data groups described below are only those data groups defined by CygNet in sample device template file(s). Your template(s) might not include some of the data groups described below. Device template files exist to enable users to customize device configurations; however, CygNet is not responsible for changes made by users.

For information about data group definitions and device template files, see Device Template Files.

For more information about data group dependencies, see Data Group Dependencies.

Note: Do not add, delete, or modify data groups in templates for this EIE. You can map UDCs to data group elements within the template, but do not otherwise alter the data group elements.

Data Group Type Usage Notes

Accum

"Accumulator"

AccumCmd

"Accumulator Command" is used to reset the accumulator channel to zero.

AgaGlobal

"AGA Global Data"

AlgHist

"Analog History Record" enables the retrieval of analog history for up to 60 individual records.

"Analog History Record" data groups can be added individually using the Add button on the Data Group page of the remote device editor.

They can also be added by means of the "Analog History Config" data group above.

In either case, a finite number (60) of "Analog History Record" data groups can be added and each record number can only be used once.

Record numbers are displayed as Ordinal numbers in the Data Groups field of the remote device editor.

AlgHistCfg

Retrieved analog history data groups populate the Analog Configuration Summary field. You have several options for adding and viewing analog history records.

To add a single history record, highlight a record in the Analog Configuration Summary field and click Add Data Group.

To view this single record in the Analog History Config dialog box Analog History field, click Get from RTU in the Analog History area, then specify the analog history request date and click OK. Double-click the record to open its corresponding dialog box.

To add all retrieved analog history data groups at once, click Add Data Groups in the Analog Configuration Summary area. View these newly added data groups in the Data Groups field of the remote device editor.

AlgInpCal

"Analog Input Calibration" is used to retrieve RTU calibration settings from corresponding analog input records for up to 60 individual records.

Record numbers are displayed as Ordinal numbers in the Data Groups field of the remote device editor.

AlgOutput

"ProControl - Analog Output"

Analog

"S/W Analog Input Record"

AutoAdjRec

"Auto Adjust/Auto Recovery" enables collection of plunger lift operations information in order to optimize plunger lift operation.

Only supported on field devices that use firmware v3:2.0 and above.

In effect, this data group is a composite of "Auto Recovery" and "Auto Tune" for firmware versions later than those that support "Auto Recovery" and "Auto Tune."

AutoRecov

"Auto Recovery" is only supported on field devices that use firmware v3:1.x.

AutoTune

"Auto Tune" is only supported on field devices that use firmware v3:0.1 to v3:0.8.

BasicPoll

"Basic Poll"

CommPort

"Comm Port Settings" enables you to define communication port settings on the associated RTU. These settings are different from those defined by the communication device configured for the CygNet host.

Composite Data Group

The "Composite Data Group" is a special kind of data group that enables you to combine any data group element(s) from any data group(s) as long as they are in the same template as the composite data group.

The "Composite Data Group" does not appear in a remote device editor unless it has been defined in the relevant device template file.

See Composite Data Group.

CryoutSnF

"Miscellaneous Setup Data"

CtdConfig

"CTD Configuration" is required to supply data to the "FMS Configuration" data group.

CtdEvents

"CTD Daily Events" is required to supply data to the "FMS Alarms" and "FMS Events" data groups. It provides visibility of native data used by CygNet Measurement.

CtdHistory

"CTD Daily History" is required to supply data to the "FMS Daily PM History" and "FMS Hourly PM History" data groups. It provides visibility of native data used by CygNet Measurement.

The "CTD Daily History" data group interprets request dates and times in contract time, and it marks the returned data in contract time.

DateTime

"Date, Time, End-of-Day" is used to read a field device's time and to write a contract hour and host time to that field device. No other date and time edits can be made.

DOutput

"ProControl - Digital Output"

DOutputCmd

"Digital Output Command"

FlowCtrl

"ProControl - Flow Control" is initialized even if Flow Control Override is specified. However, if Flow Control parameters are exceeded, Flow Control Override settings take effect.

The Flow Control Method option you select affects the two input fields beneath Deadband. For instance, if you select Pressure Differential, two new input fields generate: High Pressure Analog # and Low Pressure Analog #.

Values for the Actuation Time field can be discrete for DO or percent for AO. Type the numerical value that corresponds to the output type for the associated valve. For instance, if the associated valve is AO and you type 15 in this field, the value is understood as 15% not 15 msecs.

"FMS Alarms"

  • CTD Configuration

  • RTU Configuration

"FMS Events"

FmsHistDGm

"FMS Daily PM History" interprets request dates and times in device time, and it marks the returned data in device time.

Dependencies:
  • CTD Daily History
  • FMS Configuration

FmsHistHGm

"FMS Hourly PM History" interprets request dates and times in device time, and it marks the returned data in device time.

Dependencies:
  • CTD Daily History
  • FMS Configuration

FossGaul

"Foss Gaul Casing Sway Data" is supported with 3.3.8 and above firmware. It retrieves data that is used in Foss and Gaul modeling.

ModStatus

"ProControl - Module Status" fields are as follows:

  • PC: Production Control ON/OFF
  • VC: Production Volume Control ON/OFF
  • SD: PSD/TSD ON/OFF
  • OP: Optimization ON/OFF
  • IN: Intermitter ON/OFF
  • LC: Lift Control ON/OFF
  • FC: Flow/Override Control ON/OFF
  • PVI: Production Valve Initialized YES/NO

Password

"Password"

"Power Switching Setup" - this feature is only available with v3:0.8 or later firmware.

PrdCtrlSt

"Production Control Status" enables the reading of numerous production control data items in a single location.

This feature is only available with v3:0.5 or later firmware.

PrdHist

"Production History"

ProControl

"ProControl - Valve Data"

ProVlvOnOf

"Production Control Status On/Off"

ProVlvOpCl

"Production Valve Open/Close" enables you to write an open valve or close valve command to an associated flow computer.

This command only works if the production control module for the specified flow computer is configured and if there is no emergency shutdown condition preexisting.

ProVolume

"ProControl - Production Volume"

RtuConfig

A valid "RTU Configuration" data group must be retrieved before other data groups can be retrieved or sent.

You do not have to request the "RTU Configuration" data group every time you retrieve from or send to another data group if the last "RTU Configuration" retrieval was successful.

"Analog Output Setpoint Cmd"

Shutdown

"Status/Digital Input Record"

SwAO

For field devices using firmware v3.3.8+, the "S/W Analog Output Record" data group includes two fields not available in previous versions. The fields are MBScale and MBOffset. If you are using firmware v3.3.8+, you must upgrade to CygNet v8.2.1+ and get the latest device template file changes for SwAO. These changes include definitions of message and firmware version, as well as the two new data group elements.

SwDI

"S/W Digital Input Record"

SwDO

"S/W Digital Output Record"

Tank

"Tank/Totalizer Record" retrieves tank/totalizer information. The data group elements that are retrieved are based on firmware version.

Trend

"Trend Data" - use the Trend Tool to map trend UDCs/points to a trend screen for display.

TrendEdit

"Trend Edit" records changes made via the Modify Trend button on the Trend Configuration Data group.

TrendList

"Trend Configuration" - in the Trend List area, click Get from RTU to retrieve all the trends currently configured on the RTU. This adds Trend Data data groups to the remote device editor for each trend currently configured on the RTU.

Double-click individual trends to generate a Trend Data dialog box for the specified trend.

Use the options in the Selected Trend(s) area to manipulate a selected trend.
