Modbus EFM EIE

Important: The Modbus EIE has been superseded by the Modbus EFM EIE. See the CygNet Upgrade Procedure for more information. If you currently have Modbus EIEs deployed, refer to Modbus EIE documentation released in earlier versions of the CygNet Help.

The Modbus EFM EIE supports connectivity to Modbus RTU devices using the standard Gould/Modicon Modbus protocol as well as Enron extensions for single- and multi-register read/writes of 32-bit integer/floating point and string values. Modbus RTU, TCP, and ASCII protocols are supported. The selection of which protocol is a property of the remote device. In addition to support for common Modbus data groups, the Modbus EFM EIE also supports Enron events and history, and FMS data groups.

Modbus EFM EIE data groups are highly configurable and depend on you to define each to suit your unique needs. This enables you to create and alter interface templates for a range of Modbus devices, which vary widely in their configuration requirements. FMS data groups depend on other data groups to be defined in order for FMS transactions to be completed. The "Composite Data Group" is supported by this EIE.

Custom Modbus data groups can be added and deleted. Enron data groups must contain the correct number and types of data group elements per the Enron EFM standard. For applicable data group elements, units must be defined in the template. See your field device's manufacturer documentation for supported unit types. UDCs can be mapped to data group elements.
