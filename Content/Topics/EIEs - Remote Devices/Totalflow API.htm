Totalflow API

The Totalflow driver requires the ABB Totalflow toolkit API (tcidll.dll or tcidll64.dll) to formulate messages to and parse responses from a field device. The files do not require registration, but the Totalflow EIE cannot be registered without the correct version of these files. If the correct version of either file is not found, Totalflow devices do not load in the UIS and the UIS log file notes the incompatibility.

The .dll files are included in the CygNet SCADA product image and are copied to the applicable folder by the CygNet installer.

Note: For devices running 6400, 6700, and API firmware, only host time can be sent from the device using the "Command Message" (CmdMsg) data group. The Time Zone page on the Device Editor must be set to the default "Always Use Computer Time Zone". If another time zone is selected, a command message will not be sent.

tcidll.dll

32-bit Version (tcidll.dll)

The 32-bit Totalflow driver requires ABB's 32-bit toolkit API (tcidll.dll). The correct version of the .dll must be installed in the Windows\System32 folder (or Windows\SysWOW64 folder if running a 64-bit operating system) on the host server.

Note: For information about overriding the version safeguard for a tcidll.dll, see UIS Configuration File Keywords.

64-bit Version (tcidll64.dll)

The 64-bit Totalflow driver requires ABB’s 64-bit toolkit API (tcidll64.dll). The correct version of the .dll must be installed in the Windows\System32 folder (on a 64-bit operating system) on the host server.

Contact ABB for more information about its 64-bit implementation.
