Device Template File Hierarchy

The structure of a CygNet device template file is hierarchical; elements are nested within other elements, creating structural relationships. While the structure of a specific device template file depends on many variables, available component elements fit into a defined position relative to other component elements.

The following tables are intended to help you visualize the structural relationships between component elements. Not all of the component elements listed here are used in all EIEs. Use the following example hierarchies as a pattern to follow when editing existing device template files.
