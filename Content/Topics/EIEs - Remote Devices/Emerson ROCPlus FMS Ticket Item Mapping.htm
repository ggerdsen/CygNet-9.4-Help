FMS Ticket Item Mapping

In order to map transactional history points to measurement ticket items, you must know the name of the applicable points and their corresponding items. Such mapping occurs by using the "FMS Liquid Ticket" data group.

In a few cases, transactional history points have no corresponding measurement ticket items, or they have corresponding measurement ticket items but their enumerated values might cause confusion. If transactional history points have no corresponding measurement ticket items, you must write a custom program on the field device to accommodate the discrepancy. In either case, the enumerated values associated with the measurement ticket items must be exact. If they are not, the measurement application rejects the transaction.

See the following subsections below for more information:

  • Ticket Items with Unknown Points
  • Ticket Items with Confusing Enumerations

For more information, see EFM Data Items.
