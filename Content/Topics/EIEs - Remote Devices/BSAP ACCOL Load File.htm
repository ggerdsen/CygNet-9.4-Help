ACCOL Load File

The ACCOL load file defines the data representation and I/O for BSAP 33xx devices. Multiple load files can be added to a device to choose between, but only one is active (selected) at a time.

Adding an ACCOL Load File to a Device

  1. Show the properties of the device in the DDS.
  2. Click the Data Group page.
  3. Click the Rtu Config data group, then click View Data.
  4. Click Options, and select ACCOL Load from the drop-down menu.
  5. Click Add to add a new file. The file can be added by selecting its binary version (.acl) or text version (.acc). Selecting one version will automatically insert the other as well if it resides in the same directory.
  6. In the ACCOL Load dialog box, click the Browse button next to either the Binary load file (ACL) or Source file (ACC) fields to select a file, add any desired comments about the file in the Comment field, then click OK.
  7. To use the file for configuration, highlight the file and click Use for Config.
  8. Click Close to exit the ACCOL Load dialog box.
  9. Close the RTU Config dialog box and the Device Editor for the changes to take effect.

Selecting an ACCOL Load File to Use for Configuration

  1. On the Data Group page of a BSAP Device Editor, select the Rtu Config data group.
  2. Click View Data.
  3. Click Options, and select ACCOL Load from the drop-down menu.
  4. In the Available files field, highlight the desired file and click Use for Config.
  5. Click Close to exit the ACCOL Load dialog box.
  6. Close the RTU Config dialog box and the Device Editor for the changes to take effect.
