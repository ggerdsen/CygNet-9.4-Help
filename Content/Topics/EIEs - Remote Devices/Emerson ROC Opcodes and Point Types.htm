Opcodes and Point Types

Note: Do not confuse operator identification codes ("opcodes") with either the "Configurable Opcode Settings" or "Configurable Opcode Data" data group. Operator identification codes are special codes used by Fisher messages to define the action they perform and/or the structure of the message of which they are part.

Native CygNet data groups for Emerson ROC EIEs are predefined to use appropriate messages, opcodes, and TLPs to interact with a field device. Most of the time, the exact combination of these items is irrelevant to the proper use of a data group because of the predefined nature of the messages.

However, it is possible to create your own unique data group, using whatever allowed opcode and TLP combination you require, by adding and configuring a "Generic TLP Data" data group. In effect, you can use a "Generic TLP Data" data group to access and organize any TLP data from any supported Emerson ROC field device. For more information, see Generic TLP Data Group.

For more specific information about opcodes and point types, see the appropriate manufacturer-provided documentation.
