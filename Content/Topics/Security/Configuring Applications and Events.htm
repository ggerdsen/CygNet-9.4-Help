Configuring Applications and Events

To add Applications and Events to the ACS, you must be a Security Supervisor.

You can add an Application and Event while adding a user or you can add them separately. The steps below outline adding them in lieu of adding a user. An Application will not appear in the ACS window in CygNet Explorer unless one or more permissions have been assigned to it.

To Add an Application

  1. Using CygNet Explorer, click on the ACS in the service hierarchy.
  2. Click the Permissions page or right-click in the service window and click View Permissions.
  3. Right-click in the service window and select Apps List to show the Applications List dialog box.
  4. Click New to show the New Application dialog box.
  5. On the Events and Permissions page:
  1. In the Application text box, type the name of the Application. See Security Reference for a list of default service Application names.
  2. In the Description text box, type a description for the Application.
  1. Add Events and Permissions as described in To Add an Event to an Application or click OK to save the Application. 
  2. Click Close to close the Applications List dialog box.

To Copy an Application

When you copy an Application, all of its Events and Permissions are copied, too.

  1. Using CygNet Explorer, click on the ACS in the service hierarchy.
  2. Click the Permissions page or right-click in the service window and click View Permissions.
  3. Right-click in the service window and select Apps List to show the Applications List dialog box.
  4. Click on the Application you want to copy and click Copy. All Events and Permissions will be copied.
  5. On the Events and Permissions page in the New Application dialog box:
  1. In the Application text box, type the name of the new Application. See Security Reference for a list of default service Application names.
  2. In the Description text box, type a description for the new Application.
  1. Click OK.
  2. Click Close to close the Applications List dialog box.

To Delete an Application

When you delete an Application, all of its Events and Permissions are deleted, too.

  1. Using CygNet Explorer, click on the ACS in the service hierarchy.
  2. Click the Permissions page or right-click in the service window and click View Permissions.
  3. Right-click in the service window and select Apps List to show the Applications List dialog box.
  4. Select the Application to be deleted and click Delete.
  5. Click OK on the Delete Application ACS dialog box.
  6. Click OK to proceed with the deletion.
  7. Click Close to close the Application List dialog box.

To Add an Event to an Application

  1. Using CygNet Explorer, click on the ACS in the service hierarchy.
  2. Click the Permissions page or right-click in the service window and click View Permissions.
  3. Right-click in the service window and select Apps List to show the Applications List dialog box.
  4. Select the Application to which you want to add the Event and click Edit.
  5. On the Events and Permissions page:
  1. In the Events area, click New.
  2. In the Event text box, type the Event name. See Security Reference for a list of default service Event names.
  3. In the Description text box, type a description for the Event.
  4. Click OK.
  1. Repeat step 4 to continue to add events.
  2. Click OK to save the Application.
  3. Click Close to close the Applications List dialog box.
