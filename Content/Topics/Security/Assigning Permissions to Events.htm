Assigning Permissions to Events

To manage permissions in the ACS you must be a Security Supervisor or an Application Administrator.

When you add a permission to the ACS, you must specify the Application, the Event, the ID Type, and the Authorization Level. See Permissions for more information about user authentication, user ID types, and authorization levels.

To Add a Permission

  1. Using CygNet Explorer, click on the ACS in the service hierarchy.
  2. Select the Permissions page or right-click in the service window and click View Permissions.
  3. Right-click on the Application for which you want to add the permission and click New Permission.
  1. In the Type text box, select the Security ID Type.
  2. In the User text box, type the ID of the user or group. Wildcards can be used only for a User ID. If you are adding an ID for a CygNet Group, be sure to type the exact Group ID.
  3. In the Access Level menu, select the authorization level for the user or group. See Security Reference for a list of Authorization Levels and Tasks for each service.
  4. Click OK to close the Permission Properties dialog box or click View App… to see the Properties for Application dialog box.

To Edit a Permission

  1. Using CygNet Explorer, click on the ACS in the service hierarchy.
  2. Select the Permissions page or right-click in the service window and click View Permissions.
  3. Right-click on the entry and click Permission Properties.
  4. Edit the properties as necessary.
  5. Click OK to close the Permission Properties dialog box or click View App… to see the Properties for Application dialog box.
  6. You can also view Audit History… if desired.

To Delete a Permission

  1. Using CygNet Explorer, click on the ACS in the service hierarchy.
  2. Select the Permissions page or right-click in the service window and click View Permissions.
  3. Right-click on the entry and click Delete Permission.
  4. Click Yes to confirm the deletion.
