History Data

The VHS is the repository of historical point values passed from the current value services. History data can be viewed using CygNet Explorer, CygNet Trend, CygNet Studio/Vision, and CygNet ActiveX controls. It can be exported using CygNet History Export, CygNet Import Export utility, and CygNet Trend.

Each historical value stored for a given point contains the following items:

  • Timestamp
  • Value (16-byte string)
  • Point State
  • Base status bits
  • User status bits
Historical Data for CYGSITE8_WELL01_PDIFF

History data is stored in the VHS in its raw format. The raw format is the actual timestamp and value as reported; the service does not thin the data over time. CygNet’s trending tool, the CygNet Export utility, and the CygNet Excel Export application allow you to view history data in a rolled-up format rather than its raw format. For example, if you trend all of the raw values for a point for the past three years, the amount of data may be unwieldy for analyzing the graph. If you thin the data to an average value for each day, the data would be easier to decipher. See History Rollups for more information about rollup types.

Notes:

  • See Point Status Bits for more information about Base Status Bits and User Status Bits.
  • Alarm and notification history are stored in the Alarm Event Logging Service (ELSALM).
