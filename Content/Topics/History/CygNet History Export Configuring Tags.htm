Configuring History Export Tags

The Tags page specifies the IDs of tags for which you wish to export data. The Tag list can be built in the export definition or imported, or the export can be associated with a source file that contains the list of IDs.

The following table lists and describes fields found on the Tags page of the CygNet History Export utility.

Tag List Option Description

Enter Tag List

Tags, or points, are exported in the order in which they are listed in the tag list or the flat file. You can use the list order buttons to move an item up or down in the list.
Choose Tag

To Build the Tag List

  1. To build the tag list in the export, click Choose Tag to display the Select Point dialog box.
  2. Tags can be selected by Point ID, Long ID, or Facility/UDC. When selecting by Facility/UDC, you can select a specific facility in the Facility ID field or select all facilities. When all facilities are selected, a wildcard (*) is used for the facility identifier. In such cases, the data will be exported for all facilities that have a tag using the selected UDC(s).
  3. To add a tag identifier to the Selected List, select identifier type (Point ID, Long ID, or Facility/UDC), highlight the identifier and click Add.
  4. To remove a tag identifier from the Selected List, highlight the identifier in the Selected List and click Remove.
  5. Once the Selected List is complete, click OK.

If any of the tags are identified by Facility/UDC, you must enable the Use Point Cfg Svc to Resolve Facility/UDCs to Point IDs option.

Import Tags

To import the tag list from a source file (.txt or .csv), click the import tags button. The tags in the import file must be listed one per row as shown in the example below.

See Tag String Formats in the Points section for more information.

Read Tags from File before Export

To read the tag list from a source file (.txt or .csv), enable the Read Tags from File before Export option. Type the file path/name in the Tag File Name field or click the File browse button to select it. Note that when you associated the export with a source file, the rollup type, if applicable, applies to all of the tags. The tags in the source file must be listed one per row.
Use Point Cfg Svc to Resolve Facility/UDCs to Point IDs

Use this option of any of the tags you want to export are identified by Facility/UDC.

Rollup

If you have rollups enabled (on the Values page), you can specify the rollup type for each tag in the tag list. For Rollup Type definitions and information about how history values are used in rollups, see History Rollups.
