Notification Record Types

A GNS notification record can be one of four types:

  • Event
  • Group
  • Address
  • Unspecified

The purpose of the different record types is to help you structure the database in an easy to manage format. Each type has a specific set of rules. For example, only Event or Unspecified record types can be associated with an alarm. The record types and their associated rules are described below.

Event Record Type

An Event record type has the following properties:

  • Can be associated with an alarm in the point configuration record.
  • Contains the alarm Set and alarm Clear messages.
  • Must have at least one message.
  • Must have at least one recipient. Ideally the recipient(s) should be a Reference Address to a Group record type or an Address record type.
  • Has security settings for acknowledgment and deletion of messages in the queue. See General Notification Service (GNS) security reference.
  • Resend rules (acknowledgment, report return to normal, resend, and blackout options) can be propagated down to all records chained to the Event record type via reference addressing.
  • Messages can be previewed by clicking Play. Note that tokens are not replaced with data during the preview.
  • A test notification can be sent to all recipients using the Send Test Notification button.
  • The GNS ID associated with a Reference Address or Next GNS ID to Notify can be only a Group or Address record type.
  • See Defining an Event Record and Configuring Messages.

Group Record Type

A Group record type has the following properties:

  • Must have at least one recipient. Ideally recipient(s) should be Reference Addresses to Address record types; however, they can be raw addresses (that is, any notification type).
  • Cannot contain alarm Set or alarm Clear messages. When an Event or Unspecified record type is changed to a Group record type, you have the option of clearing the messages or retaining them. Messages are disabled.
  • Resend rules (acknowledgment, report return to normal, resend, and blackout options) can be propagated down to all records chained to the Group record type via reference addressing.
  • Cannot be associated with an alarm in the PNT.
  • The GNS ID associated with a Reference Address or Next GNS ID to Notify can be only a Group or Address record type.
  • See Adding an Address Record to a Group Record.

Address Record Type

An Address record type has the following properties:

  • Only one address can be included in each record. If a recipient has more than one address, a separate Address record type must be created for each address. Use a Group record type to group multiple addresses for a recipient.
  • Cannot contain alarm Set or alarm Clear messages. When an Event or Unspecified record type is changed to an Address record type, you have the option of clearing the messages or retaining them. Messages are disabled.
  • Cannot be associated with an alarm in the PNT.
  • See Configuring an Address Record.

Unspecified Record Type

An Unspecified record type has no rules. It can be associated with an alarm in the PNT.
