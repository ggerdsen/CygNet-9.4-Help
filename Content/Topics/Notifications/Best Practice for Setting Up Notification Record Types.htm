Best Practice for Setting Up Notification Record Types

The ideal setup for GNS records is as follows:

  • The recipients of an Event record should be Reference Address notifications to Group records.
  • The recipients of a Group record should be Reference Address notifications to Address records and other Group records.
  • The Address record contains a recipient’s raw address (email address, telephone number, etc.).
  • Unspecified records should only be used while transitioning to a new GNS model.

The advantage of using the Address record for the recipient's raw address is two-fold:

  1. The first advantage is maintenance. By strictly adhering to the use of Address records, an address change would require the administrator to change only a single record.
  1. The second advantage is data entry. When an address is entered in multiple records, the chance of data entry error is increased. Using Address records helps to isolate errors.

Group records have similar advantages to Address records. When using a Group record, you can change the members of the group in a single record rather than searching in multiple records for a recipient.

Event and Group records also allow resend rules (acknowledgment, resend, and blackout settings) to be propagated down to all records chained to them via reference addressing. Therefore, if one of these rules changes, it has to be changed only at one level.
