Acknowledging Notifications

Any type of notification can require an acknowledgment. The acknowledgment settings are configured as part of the Address record. Each notification requiring acknowledgment is assigned an acknowledgment code or ID (Ack ID), which is sent as part of the notification. One acknowledgment code exists for a "tree" of notifications that might spawn from an original alarm.

The notification will be sent repeatedly (number of retries and delay between retries is configurable). Once the number of retries is exhausted, if the recipient does not acknowledge the notification it will be sent to the GNS ID specified in the Next GNS ID to Notify field. If Acknowledgment Required is checked, the number of Retries if not acknowledged must be greater than zero or a Next GNS ID to Notify must be specified.

The method of acknowledgment depends on the type of notification. The following table summarizes the methods of acknowledgment available for each type of notification.

Acknowledging a notification of any type removes all items with the same Ack ID from the GNS Resend Queue. You must have at least Level 3 authorization (Add) for the GNS service’s QUEUE event to acknowledge a notification.

Acknowledging Any Notification Via the GNS Queue

Notifications of any type can be acknowledged by clicking Ack in the GNS Queue Viewer. See Managing the GNS Queue for information about acknowledging notifications using the GNS Queue Viewer.

Acknowledging Any Notification Via Email

Notifications of any type can be acknowledged by sending an email to the GNS. Email acknowledgment requires some additional system configuration. See Acknowledging Notifications Via Email for more information.

Acknowledging Email and SMS Notification Via SMS

Email and SMS notifications can be acknowledged by sending an email or an SMS back to the GNS. Both acknowledgment methods require some additional system configuration. See Acknowledging Notifications Via Email for more information.

Acknowledging Voice Address and WAV Address Messages

If the notification type is a Voice Address or WAV Address message the recipient has the option of acknowledging the callout while on the line with the system. Other methods to acknowledge a voice notification include:

  • dial-in to the system and enter the Ack ID (this requires that the host have a Dialogic board or a SIP Server). See Voice Address Messages for more information about configuring notification acknowledgment for Voice Address notifications.
  • use CygNet Explorer and acknowledge the message in the GNS Queue.
  • send an acknowledgment email which must include the Ack ID and PIN. See Acknowledging Notifications Via Email for more information.

The GNS can be configured to append the RESPONSE_PHONE_NUMBER to messages that require acknowledgment.

Note: You cannot acknowledge a Voice Address or WAV message via an SMS.
