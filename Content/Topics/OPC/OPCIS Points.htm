OPCIS Points

OPCIS points are stored in the Point Service (PNT) and are linked to points in the OPC server using the External ID property on the Point Reference page of the PNT editor. There are two types of OPCIS points:

  1. OPCIS Data Points
  2. OPCIS Group Status Points

OPCIS Data Points

An OPCIS data point is a point that represents an OPC server data group element. See OPCIS Data Points for more information.

OPCIS Group Status Points

An OPCIS group status point is a point that defines or monitors a property of an OPCIS facility. OPCIS group status points serve several purposes:

  • They can be used to monitor statistics of the OPC server, OPCIS groups, and the OPCIS. Items that can be monitored include status of the OPC server, its start time, values received, group update rates (configured and actual), number of OPCIS groups, etc.
  • They can be used to control the update rate of an OPCIS Facility Group, force health checks, reregister points, and to get all values for a group.

An OPCIS group status tag is just like any other tag in the PNT insomuch as it can be configured for alarms, notifications, and history. The properties that make an OPCIS group status tag unique are its facility (the name of the OPCIS group or the OPCIS) and its OPCIS-specific UDC.

An OPCIS has at least one group — the default group. The facility of this group is OPCDEFAULTGROUP. OPCIS group status points can be added for the default group or any custom group.
